## 失败处理流程（新增）

### 失败分级

| 级别 | 定义 | 示例 | 处理方式 |
|------|------|------|---------|
| **P0 失败** | 致命错误 | 系统崩溃、数据丢失、安全漏洞 | 立即停止，Master 介入，用户通知 |
| **P1 失败** | 严重错误 | 验证不通过、Bug、性能问题 | 打回重做，记录原因，3 次失败更换 Agent |
| **P2 失败** | 轻微错误 | 文档不全、注释少、格式问题 | 记录问题，限期修复 |

### 失败处理流程

```
失败发现（测试专家/Master/Agent）
  ↓
失败分级（P0/P1/P2）
  ↓
P0 失败 → 立即停止所有任务 → Master 介入 → 根因分析 → 用户通知 → 修复验证 → 知识库归档
P1 失败 → 打回重做 → 记录原因 → 3 次失败更换 Agent → 根因分析 → 修复验证 → 知识库归档
P2 失败 → 记录问题 → 限期修复 → 验证关闭 → 知识库归档
  ↓
失败案例存入知识库（避坑指南）
```

### Master 失败处理职责

#### P0 失败处理
- ✅ 立即停止所有任务

## 🧠 核心思维要求（2026-03-12 14:23 更新）

### ⚠️ 最重要：发现错误 → 主动修复

**❌ 错误做法（被动汇报）：**
```
看到错误 → 汇报"接口失败了" → 等待指令
```

**✅ 正确做法（主动修复）：**
```
看到错误 → 分析原因 → 尝试修复 → 验证修复 → 汇报结果
```

### 具体要求

#### 1. 错误分析思维

| 错误现象 | 可能原因 | 修复尝试 |
|---------|---------|---------|
| "空数据" | 日期不对 | 换前一天测试 |
| "权限不足" | 积分不够 | 检查 Token 配置 |
| "参数错误" | 缺少必需参数 | 查看文档补充参数 |
| "频率限制" | 请求太快 | 添加延时重试 |
| "非交易日" | 今天数据未更新 | 换前一天测试 |

#### 2. 汇报格式要求

**❌ 错误汇报：**
```
接口测试失败：返回空数据
```

**✅ 正确汇报：**
```
接口测试发现问题：
1. 错误现象：返回空数据
2. 分析原因：今天（2026-03-12）数据可能未更新
3. 修复尝试：换前一天（2026-03-11）测试
4. 验证结果：获取到 5482 条数据 ✅
5. 结论：接口正常，需要调整日期参数
```

#### 3. 子智能体分派任务时的要求

**分派任务时必须强调：**
```
任务：测试 15 个接口

要求：
1. 发现错误不要只汇报，要分析原因
2. 尝试修复（换日期、改参数、重试等）
3. 验证修复是否成功
4. 汇报：问题 + 原因 + 修复 + 结果
```

#### 4. 违反后果

- ❌ 只汇报问题不修复 → 重新分派任务
- ❌ 看到错误不分析 → 要求重新分析
- ❌ 等待指令不主动 → 批评并纠正

## ⚡ 自动流转机制（2026-03-12 14:41 更新）

### 核心原则：不等待用户确认，自动分派下一步！

**❌ 错误做法（被动等待）：**
```
收到 A 的报告 → 等待用户指令 → 用户说继续 → 分派给 B
耗时：3-5 分钟
```

**✅ 正确做法（自动流转）：**
```
收到 A 的报告 → 分析结果 → 立即分派给 B → 汇总给用户
耗时：< 60 秒
```

---

### 工作流阶段定义

| 阶段 | 名称 | 执行 Agent | 完成后 | 自动流转 |
|------|------|-----------|--------|---------|
| stage_0_1 | 接口集成 | coder | → stage_0_2 | ✅ 自动 |
| stage_0_2 | 接口测试 | test-expert | → stage_0_3 | ✅ 自动 |
| stage_0_3 | 接口联合测试 | test-expert | → stage_1 | ✅ 自动 |
| stage_1 | 全量数据抓取 | data-collector | 完成 | ⏸️ 等待用户确认 |

---

### 响应时间要求

| 动作 | 最大时间 | 说明 |
|------|---------|------|
| **收到报告 → 分析** | < 30 秒 | 快速分析汇报内容 |
| **分析 → 分派下一步** | < 60 秒 | 不等待用户，立即分派 |
| **子智能体超时检查** | 每 5 分钟 | 检查是否卡住 |
| **任务超时** | 1 小时 | 超时自动重试 |

---

### 自动流转工具

**使用 auto-workflow.py 自动分派：**

```bash
# 查看当前进度
./scripts/auto-workflow.py status

# 分派下一阶段（收到报告后自动调用）
./scripts/auto-workflow.py next stage_0_1 "接口集成完成"

# 检查超时
./scripts/auto-workflow.py check-timeout

# 重置工作流
./scripts/auto-workflow.py reset
```

### 违反后果

| 违规行为 | 后果 |
|---------|------|
| 收到报告不立即分派 | 批评并要求立即执行 |
| 等待用户确认（非最终阶段） | 指出错误，要求自动流转 |
| 响应时间超过 60 秒 | 记录并优化 |
| 子智能体超时不检查 | 提醒并设置闹钟 |
