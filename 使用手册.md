## 🛠️ 安装与配置

### 系统要求

### 依赖安装

```bash

pip install pyarrow matplotlib akshare pyyaml colorama

# 完整依赖一键安装
pip install -r requirements.txt
```

### 配置文件说明

系统支持两种配置方式：

#### 方式 1：Python 配置文件（fetch_data_optimized.py 头部）

```python
# 核心运行配置
AUTO_RUN_MODE = "每日选股"      # 运行模式
STRATEGY_TYPE = "打板策略"      # 策略类型
ALLOWED_MARKET = ["主板"]       # 允许板块

# 时间配置
START_DATE = "2020-01-01"       # 开始日期
END_DATE = "2026-03-11"         # 结束日期

# Tushare 配置
TUSHARE_TOKEN = "your_token"    # 替换为你的 Token
```

#### 方式 2：YAML 配置文件（config.yaml）

```yaml
core:
  auto_run_mode: "每日选股"
  strategy_type: "打板策略"
  allowed_market: ["主板"]

server:
  tushare_token: "your_token"
  log_level: "INFO"
```

**优先级：** 环境变量 > config.yaml > Python 配置文件

### 环境变量配置

```bash
# Linux/Mac
export QUANT_AUTO_RUN_MODE="每日选股"
export QUANT_TUSHARE_TOKEN="your_token"
export QUANT_LOG_LEVEL="DEBUG"

# Windows (PowerShell)
$env:QUANT_AUTO_RUN_MODE="每日选股"
$env:QUANT_TUSHARE_TOKEN="your_token"
```

## 📊 运行模式说明

### 模式对比表

| 模式 | 用途 | 耗时 | 数据需求 | 适用场景 |
|------|------|------|---------|---------|
| **抓取 + 回测** | 全量数据抓取 + 回测验证 | 2-8 小时 | Tushare API | 首次使用、策略验证 |
| **仅服务** | 启动 API 服务 | <1 分钟 | 本地数据 | 提供数据接口 |
| **仅回测** | 使用本地数据回测 | 10-60 分钟 | 本地缓存 | 快速验证策略 |
| **每日选股** | 抓取最新数据 + 选股 | 5-30 分钟 | Tushare API | 每日实盘选股 |

### 模式切换

```bash
# 切换到全量回测模式
python scripts/switch_mode.py --mode 全量

# 切换到每日选股模式
python scripts/switch_mode.py --mode 每日选股

# 仅修改配置，不重启
python scripts/switch_mode.py --mode 仅回测 --no-restart
```

## 🎯 策略配置详解

### 1. 打板策略

**策略逻辑：** 追涨停板，捕捉连板机会

**核心参数：**
```python
STRATEGY_CONFIG = {
    "打板策略": {
        "type": "link",                # 连板策略
        "min_order_ratio": 0.03,        # 最低封单比 3%
        "max_break_times": 1,           # 最大炸板 1 次
        "link_board_range": [2, 4],     # 连板高度 2-4 板
        "stop_loss_rate": 0.06,         # 止损 6%
        "stop_profit_rate": 0.12,       # 止盈 12%
        "max_hold_days": 2              # 最长持股 2 天
    }
}
```

**适用行情：** 强势市场、主线清晰  
**风险等级：** ⭐⭐⭐⭐（较高）

### 2. 缩量潜伏策略

**策略逻辑：** 涨停后缩量回调低吸，不追高

**核心参数：**
```python
STRATEGY_CONFIG = {
    "缩量潜伏策略": {
        "type": "first_board_pullback",  # 首板回调
        "first_board_limit": True,        # 必须首板
        "board_volume_growth": 1.5,       # 放量 1.5 倍
        "shrink_volume_ratio": [1/3, 1/2],# 缩量到 1/3~1/2
        "shrink_days_range": [3, 10],     # 3-10 天回调
        "pullback_support_level": 0.5,    # 支撑位 1/2
        "support_tolerance": 0.02,        # 容错±2%
        "stop_loss_rate": 0.03,           # 止损 3%
        "stop_profit_rate": 0.15,         # 止盈 15%
        "max_hold_days": 8                # 最长 8 天
    }
}
```

**适用行情：** 震荡市、轮动行情  
**风险等级：** ⭐⭐⭐（中等）

### 3. 板块轮动策略

**策略逻辑：** 捕捉行业板块轮动机会

**核心参数：**
```python
STRATEGY_CONFIG = {
    "板块轮动策略": {
        "type": "industry",            # 行业轮动
        "rotate_days": 3,               # 3 天调仓
        "stop_loss_rate": 0.05,         # 止损 5%
        "stop_profit_rate": 0.1,        # 止盈 10%
        "main_trend": True,             # 只做主线
        "fund_inflow_top": 30,          # 资金流入前 30
        "max_hold_days": 3              # 最长 3 天
    }
}
```

**适用行情：** 电风扇行情、快速轮动  
**风险等级：** ⭐⭐（较低）

## ❓ 常见问题 FAQ

### Q1: Tushare Token 如何获取？

**A:** 访问 [Tushare 官网](https://tushare.pro/) 注册账号，在个人中心获取 Token。

```python
# 修改配置文件中的 Token
TUSHARE_TOKEN = "你的 Token"
```

### Q2: 如何修改选股评分标准？

**A:** 编辑 `CORE_CONFIG` 中的评分项：

```python
CORE_CONFIG = {
    "pass_score": 12,  # 及格分
    "items": {
        "连板高度≥3 板": [2, "up_down_times >= 3", {...}],
        # 添加或修改评分项
    }
}
```

### Q3: 抓取数据太慢怎么办？

**A:** 调整并发参数：

```python
FETCH_OPTIMIZATION = {
    'max_workers': 20,              # 增加并发线程
    'max_requests_per_minute': 1500 # 提高请求频率
}
```

**注意：** 需根据 Tushare 积分调整，避免被封禁。

### Q4: 回测结果不理想如何优化？

**A:** 按以下顺序优化：

1. **调整策略参数** - 修改止损止盈、持仓天数
2. **优化评分规则** - 增加有效因子权重
3. **改进筛选条件** - 提高前置筛选标准
4. **参数进化** - 使用参数进化智能体自动优化

### Q5: 如何导出选股结果？

**A:** 选股结果自动保存到 `data/` 目录：

```bash
# Excel 格式
data/stock_pick_20260311.xlsx

# JSON 格式
data/stock_pick_20260311.json

# 评分明细
data/stock_pick_detail_20260311.xlsx
```

### Q6: 日志文件在哪里？

**A:** 日志保存在 `logs/` 目录：

```bash
logs/quant_info.log    # 信息日志（保留 30 天）
logs/quant_error.log   # 错误日志（保留 7 天）
```

### Q7: 如何查看历史回测报告？

**A:** 回测报告保存在 `data/` 目录：

```bash
data/backtest_report_20260311.json
data/backtest_chart_20260311.png
```

### Q8: 系统支持哪些数据源？

**A:** 当前支持：

- ✅ Tushare（主数据源）
- ✅ AkShare（新闻数据）
- 🔄 东方财富（开发中）
- 🔄 同花顺（开发中）

### Q9: 如何在服务器上部署？

**A:** 使用 systemd 服务：

```ini
# /etc/systemd/system/quant.service
[Unit]
Description=Quant Trading System
After=network.target

[Service]
Type=simple
User=admin
WorkingDirectory=/path/to/project
ExecStart=/usr/bin/python3 fetch_data_optimized.py
Restart=always

[Install]
WantedBy=multi-user.target
```

```bash
# 启动服务
sudo systemctl start quant
sudo systemctl enable quant
```

### Q10: 实盘交易支持吗？

**A:** ⚠️ **当前版本仅支持回测和选股，不支持实盘交易。**

实盘交易需要：

## 🔧 故障排查

### 错误代码速查

| 错误代码 | 错误信息 | 解决方案 |
|---------|---------|---------|
| E001 | Tushare Token 无效 | 检查 Token 配置 |
| E002 | 网络连接失败 | 检查网络/代理 |
| E003 | 数据格式错误 | 更新依赖库 |
| E004 | 磁盘空间不足 | 清理磁盘空间 |
| E005 | 内存溢出 | 减少并发数 |
| E006 | 端口被占用 | 修改 SERVER_PORT |
| E007 | 配置文件错误 | 检查 YAML 格式 |
| E008 | 依赖库缺失 | 运行 pip install |

### 日志分析方法

```bash
# 查看最新错误
tail -100 logs/quant_error.log

# 搜索特定错误
grep "ERROR" logs/quant_info.log

# 实时查看日志
tail -f logs/quant_info.log

# 日志分析工具
python scripts/daily_log_analyzer.py --date today
```

### 常见问题处理

#### 问题 1：抓取数据时频繁报错

**原因：** 请求频率过高，被 Tushare 限流

**解决：**
```python
FETCH_OPTIMIZATION = {
    'max_requests_per_minute': 500  # 降低请求频率
}
```

#### 问题 2：回测结果与预期不符

**原因：** 数据不完整或参数错误

**解决：**
1. 检查数据完整性：`python fetch_data_optimized.py --validate`
2. 核对策略参数
3. 查看回测日志

#### 问题 3：程序运行缓慢

**原因：** 并发数过低或硬件限制

**解决：**
1. 增加并发数（根据积分）
2. 使用 SSD 硬盘
3. 增加内存
