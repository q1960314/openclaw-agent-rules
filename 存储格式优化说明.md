### 3. 自动降级兼容（pyarrow 未安装时使用 CSV）

**实现机制：**

```python
# 依赖检测
try:
    import pyarrow as pa
    import pyarrow.parquet as pq
    PARQUET_AVAILABLE = True
except ImportError:
    PARQUET_AVAILABLE = False

# 初始化时自动检测并降级
def __init__(self, base_dir: str, storage_format: str = 'parquet'):
    if storage_format == 'parquet' and not PARQUET_AVAILABLE:
        logger.warning("⚠️  pyarrow 未安装，Parquet 存储不可用，自动降级为 CSV 格式")
        self.storage_format = 'csv'
    else:
        self.storage_format = storage_format
```

**降级策略：**
1. **Parquet 不可用时** → 自动降级为 CSV
2. **SQLite 不可用时** → 自动降级为 CSV
3. **Parquet/SQLite 保存失败时** → 自动重试 CSV
4. **Parquet/SQLite 文件不存在时** → 自动查找 CSV 文件

**兼容性保证：**
- ✅ 无需修改现有代码即可使用
- ✅ 所有存储方法都有 CSV 降级备选
- ✅ 日志明确提示降级原因
- ✅ 不影响现有 JSON 存储（保留兼容）

```bash

```

### 性能建议

1. **大数据集** → 优先使用 Parquet（列式存储、压缩率高）
2. **频繁查询** → 优先使用 SQLite（索引加速、SQL 查询）
3. **小配置文件** → 继续使用 JSON（人类可读）
4. **跨平台共享** → 使用 CSV（通用性最强）

### 迁移现有数据

如需从 CSV 迁移到 Parquet/SQLite：
```python
# CSV → Parquet
df = storage.load_csv('data.csv')
storage.save_dataframe(df, 'data', subdir='migrated')

# CSV → SQLite
df = storage.load_csv('data.csv')
storage.save_dataframe(df, table_name='stock_data')
```
