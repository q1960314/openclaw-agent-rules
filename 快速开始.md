## 🚀 第一步：环境准备（2 分钟）

### 1. 检查 Python 版本

```bash
python --version
# 需要 Python 3.8 或更高版本
```

### 2. 安装依赖

```bash
# 方式 1：完整安装（推荐）
pip install -r requirements.txt

# 方式 2：最小安装（仅核心功能）
pip install pandas numpy tushare requests tqdm flask flask-cors
```

### 3. 获取 Tushare Token

1. 访问 [Tushare 官网](https://tushare.pro/)
2. 注册账号并登录
3. 进入「个人中心」获取 Token
4. 复制 Token，备用

## ⚙️ 第二步：配置系统（1 分钟）

### 方式 1：修改 Python 配置文件

打开 `fetch_data_optimized.py`，修改以下配置：

```python
# 第 1 行附近
AUTO_RUN_MODE = "每日选股"  # 运行模式

# 第 9 行附近
STRATEGY_TYPE = "打板策略"  # 策略类型

# 第 17 行附近
ALLOWED_MARKET = ["主板"]  # 允许板块

# 第 9 节附近
TUSHARE_TOKEN = "你的 Token"  # 替换为你的 Token
```

### 方式 2：使用 YAML 配置文件

复制配置模板：

```bash
cp config.yaml config.local.yaml
```

编辑 `config.local.yaml`：

```yaml
core:
  auto_run_mode: "每日选股"
  strategy_type: "打板策略"
  allowed_market: ["主板"]

server:
  tushare_token: "你的 Token"
```

### 方式 3：使用环境变量（推荐用于服务器）

```bash
export QUANT_AUTO_RUN_MODE="每日选股"
export QUANT_STRATEGY_TYPE="打板策略"
export QUANT_TUSHARE_TOKEN="你的 Token"
```

## 🎯 第三步：运行程序（2 分钟）

### 场景 1：首次使用（全量数据 + 回测）

```bash
python fetch_data_optimized.py -m 抓取 + 回测 --start-date 2020-01-01
```

**说明：** 首次运行会抓取历史数据，耗时 2-8 小时（取决于网络和积分）

### 场景 2：每日选股（最常用）

```bash
python fetch_data_optimized.py -m 每日选股 -s 打板策略
```

**说明：** 抓取最新交易日数据，执行选股策略，耗时 5-30 分钟

### 场景 3：快速回测验证

```bash
python fetch_data_optimized.py -m 仅回测 -s 缩量潜伏策略
```

**说明：** 使用本地已有数据回测，耗时 10-60 分钟

### 场景 4：启动 API 服务

```bash
python fetch_data_optimized.py -m 仅服务
```

**说明：** 启动后端服务，提供数据接口

## 🎨 进阶用法

### 1. 切换策略

```bash

```

### 2. 选择板块


```bash

# 回测特定时间段
python fetch_data_optimized.py -m 仅回测 --start-date 2023-01-01 --end-date 2023-12-31
```

### 4. 调试模式


```bash

python fetch_data_optimized.py --config config.prod.yaml
```

## ❓ 常见问题

### Q1: 首次运行需要多久？

**A:** 取决于你的 Tushare 积分和网络：
- 积分>10000：2-4 小时
- 积分 5000-10000：4-6 小时
- 积分<5000：6-8 小时

### Q2: 选股结果在哪里？

**A:** `data/stock_pick_YYYYMMDD.xlsx`

### Q3: 如何修改选股评分标准？

**A:** 编辑 `fetch_data_optimized.py` 中的 `CORE_CONFIG` 配置

### Q4: 程序报错怎么办？

**A:** 
1. 查看错误日志：`logs/quant_error.log`
2. 检查配置是否正确
3. 确认 Tushare Token 有效
4. 查看使用手册的故障排查章节

### Q5: 如何优化抓取速度？

**A:** 修改 `FETCH_OPTIMIZATION` 配置：

```python
FETCH_OPTIMIZATION = {
    'max_workers': 20,              # 增加并发
    'max_requests_per_minute': 1500 # 提高请求频率
}
```

**注意：** 需根据 Tushare 积分调整
