# 运行模式配置说明文档

**版本：** v1.0（阶段 1.1：代码模式分离）  
**更新日期：** 2026-03-12  
**适用文件：** `fetch_data_optimized.py`

## 一、模式概述

系统现已支持 **3 种独立运行模式**，每种模式有专属配置，无需手动切换参数：

| 模式名称 | AUTO_RUN_MODE 值 | 用途 | 执行时机 |
|---------|-----------------|------|---------|
| **全量抓取模式** | `"全量抓取"` | 抓取 2020-01-01 至今的全部历史数据 | 回测前必须执行 |
| **增量抓取模式** | `"增量抓取"` | 仅抓取最新交易日数据 | 每日凌晨 4 点执行 |
| **每日选股模式** | `"每日选股"` | 使用缓存数据执行实盘选股 | 收盘后执行 |

---

## 二、模式配置详解

### 1. 全量抓取模式

**配置代码：**
```python
AUTO_RUN_MODE = "全量抓取"
FETCH_CONFIG = {
    'start_date': '2020-01-01',
    'end_date': '2026-03-11',
    'max_workers': 10,  # 保守配置
    'max_requests_per_minute': 2000,
    'save_frequency': 20,
    'verify_frequency': 100,
}
```

**配置说明：**
- `start_date`: 抓取开始日期（格式：YYYY-MM-DD）
- `end_date`: 抓取结束日期（格式：YYYY-MM-DD）
- `max_workers`: 并发线程数（保守配置，保证稳定性）
- `max_requests_per_minute`: 每分钟最大请求数（遵守 Tushare 限流）
- `save_frequency`: 每抓取 N 只股票保存一次进度
- `verify_frequency`: 每抓取 N 只股票校验一次数据完整性

**适用场景：**
- 首次使用系统，需要建立完整历史数据库
- 回测前需要全量数据支撑
- 数据损坏后需要重新抓取

**预计耗时：** 约 20-30 小时（取决于网络和 Tushare 积分）

---

### 2. 增量抓取模式

**配置代码：**
```python
AUTO_RUN_MODE = "增量抓取"
FETCH_CONFIG = {
    'start_date': '最新交易日 -30 天',
    'end_date': '最新交易日',
    'max_workers': 15,
    'max_requests_per_minute': 3000,
    'save_frequency': 50,
}
```

**配置说明：**
- `start_date`: 支持相对日期格式，自动计算为「最新交易日 -30 天」
- `end_date`: 自动识别为最新交易日
- `max_workers`: 激进配置（15 线程），提升抓取速度
- `max_requests_per_minute`: 3000 次/分钟，适合高积分用户
- `save_frequency`: 每 50 只股票保存一次（减少 IO 次数）

**适用场景：**
- 每日凌晨自动更新最新数据
- 补充最近 30 天的财务指标、龙虎榜等扩展数据
- 快速修复缺失的个股数据

**预计耗时：** 约 1-2 小时

---

### 3. 每日选股模式

**配置代码：**
```python
AUTO_RUN_MODE = "每日选股"
PICK_CONFIG = {
    'pick_time': '08:00-09:15',
    'use_cached_data': True,  # 不抓取，使用缓存
    'analyze_news': True,  # 分析隔夜消息
    'export_excel': True,
}
```

**配置说明：**
- `pick_time`: 选股时间窗口（早盘 8:00-9:15）
- `use_cached_data`: `True` 表示不抓取新数据，直接使用本地缓存
- `analyze_news`: `True` 表示分析隔夜消息面（美股/新闻/政策）
- `export_excel`: `True` 表示导出 Excel 选股清单

**适用场景：**
- 每日收盘后执行选股（17:30 后）
- 早盘前生成选股清单（8:00-9:15）
- 快速选股，无需等待数据抓取

**预计耗时：** 约 5-10 分钟

## 三、模式切换方法

### 方法 1：直接修改 AUTO_RUN_MODE

在 `fetch_data_optimized.py` 第 10 行左右，修改：

```python
AUTO_RUN_MODE = "全量抓取"  # 改为 "增量抓取" 或 "每日选股"
```

### 方法 2：通过环境变量（推荐用于自动化脚本）

```bash
# Linux/Mac
export AUTO_RUN_MODE="增量抓取"
python fetch_data_optimized.py

# Windows
set AUTO_RUN_MODE=增量抓取
python fetch_data_optimized.py
```

## 四、配置自动应用逻辑

系统会在启动时自动根据 `AUTO_RUN_MODE` 应用对应配置：

```python
if AUTO_RUN_MODE == "全量抓取":
    # 应用全量抓取配置
    FETCH_CONFIG = {...}
    START_DATE = '2020-01-01'
    END_DATE = '2026-03-11'
    FETCH_OPTIMIZATION['max_workers'] = 10
    
elif AUTO_RUN_MODE == "增量抓取":
    # 应用增量抓取配置
    FETCH_CONFIG = {...}
    START_DATE = '最新交易日 -30 天'
    END_DATE = '最新交易日'
    FETCH_OPTIMIZATION['max_workers'] = 15
    
elif AUTO_RUN_MODE == "每日选股":
    # 应用选股模式配置
    PICK_CONFIG = {...}
    STOCK_PICK_CONFIG.update(PICK_CONFIG)
```

**无需手动修改其他参数！**

## 六、常见问题

### Q1: 修改模式后配置不生效？
**A:** 检查是否修改了正确的 `AUTO_RUN_MODE` 变量（第 10 行左右），确保没有语法错误（如引号不匹配）。

### Q2: 如何自定义全量抓取的日期范围？
**A:** 直接修改 `FETCH_CONFIG` 中的 `start_date` 和 `end_date`：
```python
FETCH_CONFIG = {
    'start_date': '2023-01-01',  # 自定义开始日期
    'end_date': '2026-03-11',
    ...
}
```

### Q3: 增量抓取可以抓取更多天数吗？
**A:** 可以，修改 `start_date`：
```python
'start_date': '最新交易日 -90 天',  # 抓取最近 90 天
```

### Q4: 每日选股模式可以关闭新闻分析吗？
**A:** 可以，修改 `PICK_CONFIG`：
```python
PICK_CONFIG = {
    'analyze_news': False,  # 关闭新闻分析
    ...
}
```

## 七、最佳实践

### 1. 首次使用流程
```
1. 设置 AUTO_RUN_MODE = "全量抓取"
2. 运行程序，等待 20-30 小时完成全量数据抓取
3. 设置 AUTO_RUN_MODE = "仅回测"，验证数据完整性
4. 设置 AUTO_RUN_MODE = "每日选股"，执行首次选股
```

### 2. 日常使用流程
```
每天凌晨 4 点：AUTO_RUN_MODE = "增量抓取"（自动定时任务）
每天收盘后：AUTO_RUN_MODE = "每日选股"（手动执行）
每周一次：AUTO_RUN_MODE = "仅回测"（验证策略）
```

### 3. 数据修复流程
```
1. 设置 AUTO_RUN_MODE = "增量抓取"
2. 修改 FETCH_CONFIG['start_date'] = '最新交易日 -7 天'
3. 运行程序，补充最近 7 天数据
```
