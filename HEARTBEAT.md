# HEARTBEAT.md - master-quant 触发规则

## 触发模式：事件驱动（实时响应）

### 触发源（按优先级）

1. **用户指令** - 飞书/微信消息（最高优先级）
2. **子智能体汇报** - 任务完成/异常报告（高优先级）
3. **定时任务** - 每日数据抓取、每周总结等（中优先级）

### 无自动唤醒

- ❌ 移除每 8 分钟定时唤醒
- ❌ 无触发信号时保持待命
- ✅ 收到消息立即响应

## 子智能体汇报处理

**收到子智能体汇报时：**
1. 立即验收交付物
2. 判断是否继续下一步
3. 自动下发指令（如符合规则）
4. 记录任务进度

**超时处理：**
- 子智能体超过 10 分钟未汇报 → 主动询问
- 连续 3 次超时 → 记录异常，通知用户

## 定时任务（保留）

| 任务 | 频率 | 说明 |
|------|------|------|
| 每日数据抓取 | 工作日 21:00 | 抓取当日行情数据 |
| 每日选股 | 工作日 8:30 | 执行当日选股 |
| 每日总结 | 每日 21:30 | 生成日报 |
| 每周升级 | 周日 22:00 | 能力升级 |
| 备份 | 每日 2:00 | 系统备份 |

## 通知规则

**通知用户：**
- ✅ 全部任务完成
- ⚠️ 高风险问题
- 📊 用户询问时

**不通知：**
- 步骤正常完成
- 自动修复的问题

## 自动轮询规则（解决等待问题）

### 问题

子智能体使用 `mode: "run"` 运行，完成后会话销毁，无法主动通知 Master。

### 解决方案

**Master 必须主动轮询子智能体状态：**

```javascript
// 每 2 分钟检查一次
setInterval(() => {
  const status = subagents({action: "list"})
  if (status.recent[0].status === "done") {
    // 子智能体完成，立即继续下一步
    executeNextTask()
  }
}, 120000) // 2 分钟
```

### 轮询时机

| 时机 | 动作 |
|------|------|
| 子智能体下发后 | 启动轮询（每 2 分钟） |
| 检测到完成 | 立即继续下一步 |
| 超过 10 分钟未完成 | 发送提醒通知 |
| 超过 30 分钟未完成 | 标记异常，通知用户 |

### 当前限制

由于 OpenClaw 不支持后台定时任务，轮询需要：
- 方案 1：使用系统 cron 定时触发
- 方案 2：用户消息触发（当前方案）
- 方案 3：子智能体 webhook 通知（需要开发）

**当前采用方案 2**：通过你的消息触发 Master 继续执行。
