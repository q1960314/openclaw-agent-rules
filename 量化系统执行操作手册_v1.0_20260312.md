## 执行机制

### 1. 会话窗口管理

**规则：** 与每个子 Agent 的会话窗口保持 1 个，不关闭，重复使用

**执行方式：**
```
1. 创建子 Agent 会话窗口（例如：代码守护者 - 新接口集成）
2. 分派任务给子 Agent
3. 子 Agent 完成任务
4. 子 Agent 详细汇报（不关闭会话）
5. 下次分派任务时，使用原会话窗口
6. 循环执行
```

**会话窗口列表：**
| Agent 名称 | 会话标签 | 用途 |
|-----------|---------|------|
| 代码守护者 | 代码守护者 - 新接口集成 | 接口集成/代码修复/功能实现 |
| 测试专家 | 测试专家 - 接口测试 | 接口测试/功能测试/数据验证 |
| 数据采集员 | 数据采集员 - 数据抓取 | 数据接口调用/数据抓取 |
| 策略专家 | 策略专家 - 战法分析 | 战法库分析/策略优化 |
| 因子挖掘智能体 | 因子挖掘 - 因子分析 | 因子库分析/因子实现 |
| 回测引擎智能体 | 回测引擎 - 回测执行 | 回测框架搭建/回测执行 |
| 金融学习员 | 金融学习员 - 战法研究 | 战法收集/金融知识学习 |
| 运维监控员 | 运维监控员 - 进度监控 | 进度监控/异常告警 |
| 文档管理员 | 文档管理员 - 知识归档 | 文档整理/知识库归档 |
| 生态沉淀员 | 生态沉淀员 - 避坑指南 | 失败案例归档 |

---

### 2. 任务分派机制

**并行执行原则：**
- 同类任务可以并行分派给同一个 Agent
- 例如：15 个接口集成任务可以一次性分派给代码守护者，并行执行
- 不是串行执行（一个接一个），是并行执行（同时做 15 个）

**任务分派流程：**
```
1. 创建子 Agent 会话窗口
2. 一次性分派多个同类任务（例如 15 个接口集成）
3. 子 Agent 并行执行
4. 子 Agent 完成后统一详细汇报
5. 验收通过后分派下一批任务
```

---

### 3. 汇报机制

**任务完成汇报：** 每个任务（或每批任务）完成后立即详细汇报

**进度汇报：** 每 30 分钟汇总汇报总进度

**异常汇报：** 遇到阻塞/失败立即汇报
