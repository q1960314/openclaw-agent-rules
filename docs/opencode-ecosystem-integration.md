## 一、核心认知：我们有什么

### 1.1 用户代码资产

```
vnpy_backtest/
├── orchestrator.py      # 回测编排器（入口）
├── strategies.py        # 打板策略 + 缩量潜伏策略
├── base_strategy.py     # 策略基类 + StrategyRegistry
├── vector_engine.py     # 向量化回测引擎 v7.6
├── akshare_source.py    # 数据源
└── config_loader.py     # 配置加载

memory/
├── agent_memory.py      # 子智能体记忆接口
├── memory_manager.py    # 记忆管理器
├── knowledge_retriever.py # 知识检索
└── pressure_monitor.py  # 压力监控
```

**核心特点**：
- 策略注册机制（StrategyRegistry）支持动态加载
- 回测编排器（orchestrator）整合所有模块
- 六层记忆系统（Core + Knowledge + Working + ...）

### 1.2 主流框架启示

| 框架 | 核心理念 | 可借鉴点 |
|------|----------|----------|
| **AutoGen** | AssistantAgent + Tool | MCP 服务器集成、AgentTool 编排 |
| **MetaGPT** | Code = SOP(Team) | SOP 流程化、角色分工明确 |
| **Agno** | AgentOS 监控 | 审批流程、人在回路、审计日志 |

### 1.3 OpenCode 新能力

| 工具 | 能力 | 对现有系统的价值 |
|------|------|------------------|
| `sessions_spawn` | 动态创建子智能体 | 可替代静态配置，但用户选择保留 |
| `subagents` | 管理/终止/引导子智能体 | **关键**：解决中断问题 |
| `memory_*` | 记忆管理 | 可整合到六层记忆系统 |
| `exec` | 执行代码/脚本 | 直接运行回测 |
| `read/write` | 文件操作 | 直接查看/修改代码 |

## 四、工作流程：完整联动示例

### 4.1 策略优化完整流程

```
用户指令: "优化打板策略的评分体系"

┌─────────────────────────────────────────────────────────────┐
│ 步骤1: master-quant 接收指令                                │
│   - 检索知识库: 搜索"打板策略 评分"相关经验                   │
│   - 双向校验: 检查指令是否违反红线                            │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│ 步骤2: master-quant 拆解任务                                │
│   任务1: 分析当前评分体系 (strategy-expert)                  │
│   任务2: 挖掘新因子 (factor-miner)                           │
│   任务3: 回测验证 (backtest-engine)                          │
│   任务4: 代码更新 (coder)                                    │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│ 步骤3: 调度 strategy-expert                                 │
│   sessions_send(                                            │
│     sessionKey="agent:strategy-expert:main",                │
│     message="分析 strategies.py 中打板策略的评分体系，        │
│              找出可优化的维度"                               │
│   )                                                          │
│                                                              │
│   strategy-expert 执行:                                      │
│   - read strategies.py                                       │
│   - 分析 DEFAULT_WEIGHTS                                     │
│   - 输出分析报告                                             │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│ 步骤4: 中转调度 factor-miner (支撑层)                        │
│   master-quant 收到 strategy-expert 的分析报告后：           │
│   sessions_send(                                            │
│     sessionKey="agent:factor-miner:main",                   │
│     message="基于以下分析，挖掘新的评分因子: ..."             │
│   )                                                          │
│                                                              │
│   factor-miner 执行:                                         │
│   - 分析数据相关性                                           │
│   - 计算 IC 值                                               │
│   - 输出新因子建议                                           │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│ 步骤5: 调度 backtest-engine 回测验证                        │
│   sessions_send(                                            │
│     sessionKey="agent:backtest-engine:main",                │
│     message="用新因子回测验证，对比原评分体系"                │
│   )                                                          │
│                                                              │
│   backtest-engine 执行:                                      │
│   - exec orchestrator.run_full_backtest()                   │
│   - 对比新旧评分体系的收益                                    │
│   - 输出回测报告                                             │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│ 步骤6: 汇总结果，用户审批                                    │
│   master-quant 汇总:                                        │
│   - 分析报告                                                 │
│   - 新因子建议                                               │
│   - 回测对比结果                                             │
│                                                              │
│   发送给用户（飞书）:                                        │
│   "【策略优化方案】                                           │
│    1. 新增因子: xxx (权重10分)                               │
│    2. 调整权重: xxx                                          │
│    3. 回测收益提升: +2.5%                                    │
│    请确认是否应用？"                                          │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│ 步骤7: 用户确认后，调度 coder 更新代码                       │
│   sessions_send(                                            │
│     sessionKey="agent:coder:main",                          │
│     message="更新 strategies.py 中的 DEFAULT_WEIGHTS: ..."  │
│   )                                                          │
│                                                              │
│   coder 执行:                                                │
│   - 备份原文件                                               │
│   - write 新的 DEFAULT_WEIGHTS                               │
│   - 更新版本号                                               │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│ 步骤8: 知识库沉淀                                            │
│   sessions_send(                                            │
│     sessionKey="agent:knowledge-steward:main",              │
│     message="沉淀本次优化经验到知识库"                        │
│   )                                                          │
│                                                              │
│   knowledge-steward 执行:                                    │
│   - memory_store 经验到避坑指南                              │

└─────────────────────────────────────────────────────────────┘
```
