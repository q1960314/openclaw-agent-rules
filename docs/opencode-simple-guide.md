# OpenCode 工具升级指南

> 核心理念：工作流程不变，工具能力升级

---

## 一、不变的核心

```
用户需求 → master-quant 优化/拆解 → 分派子智能体 → 执行 → 汇报用户
```

**你的 4 个核心模式：**

| 模式 | 触发方式 | 执行者 |
|------|----------|--------|
| 1. 全量抓取 | 手动/定时 | data-collector |
| 2. 增量抓取 | 定时（每日） | data-collector |
| 3. 回测 | 手动 | backtest-engine + test-expert |
| 4. 每日选股 | 定时（盘前） | strategy-expert + coder |

---

## 二、OpenCode 带来的新能力

| 新工具 | 能力 | 应用场景 |
|--------|------|----------|
| `exec` | 直接执行代码/脚本 | 运行回测、数据处理 |
| `read` | 直接读取文件 | 查看代码、配置、日志 |
| `write` | 直接写入文件 | 更新代码、配置（需确认） |
| `memory_*` | 记忆管理 | 替代现有脚本 |
| `subagents` | 管理子智能体 | 检查状态、终止任务 |
| `sessions_spawn` | 动态创建子智能体 | 按需扩展（可选用） |

---

## 三、工具升级对照表

| 原有方式 | OpenCode 方式 | 说明 |
|----------|--------------|------|
| `python scripts/kb-search.sh "关键词"` | `memory_recall(query="关键词")` | 更简洁 |
| `python scripts/kb-update.sh` | `memory_store(text="内容")` | 更直接 |
| 通过子智能体读取代码 | `read(path="代码路径")` | master 可直接查看 |
| 通过子智能体执行脚本 | `exec(command="回测命令")` | master 可直接执行 |

## 四、简化后的工作流

### 4.1 用户需求示例：「优化打板策略」

```
┌─────────────────────────────────────────┐
│ 用户: "优化打板策略的评分体系"            │
└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
│ master-quant 执行:                       │
│                                          │
│ 1. read strategies.py (直接查看代码)     │
│ 2. 检索知识库 (memory_recall)            │
│ 3. 拆解任务                              │
└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
│ 分派任务:                                │
│                                          │
│ sessions_send(                           │
│   sessionKey="agent:strategy-expert",   │
│   message="分析打板策略评分体系，         │
│            找出优化方向"                  │
│ )                                        │
└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
│ strategy-expert 执行 → 返回分析报告      │
└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
│ master-quant 执行:                       │
│                                          │
│ exec("python run_backtest.py")          │
│ (可直接运行回测验证)                      │
└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
│ 汇总结果 → 发送飞书给用户审批            │
└─────────────────────────────────────────┘
```

### 4.2 每日选股流程

```
┌─────────────────────────────────────────┐
│ 定时触发（盘前）                         │
└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
│ master-quant 执行:                       │
│                                          │
│ 1. exec("python fetch_incremental.py")  │
│    (增量数据抓取)                        │
│                                          │
│ 2. exec("python run_selection.py")      │
│    (运行选股)                            │
└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
│ 发送选股结果到飞书                       │
└─────────────────────────────────────────┘
```
