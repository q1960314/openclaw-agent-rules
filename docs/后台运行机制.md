# 后台运行机制 - OpenClaw

## 一、问题背景

用户反馈：运行 open code 时要一直盯住，占用会话资源，无法同时沟通。

## 二、解决方案

### 方案1：使用 exec 的 background 参数

```python
# 启动后台任务
exec(
    command="/home/admin/miniconda3/envs/vnpy_env/bin/python /data/agents/master/run_backtest_v4.py",
    background=True,  # 后台运行
    yieldMs=5000      # 5秒后返回控制权
)

# 继续与用户沟通...
```

### 方案2：使用 process 检查状态

```python
# 检查后台任务状态
process(
    action="list"  # 列出所有后台任务
)

# 查看特定任务日志
process(
    action="log",
    sessionId="任务ID"
)

# 等待任务完成
process(
    action="poll",
    sessionId="任务ID",
    timeout=60000  # 最多等待60秒
)
```

### 方案3：完成后用 message 通知

```python
# 任务完成后，通过飞书通知用户
message(
    action="send",
    channel="feishu",
    target="用户ID",
    message="【后台任务完成】\n任务：回测\n状态：成功\n耗时：120秒"
)
```

## 三、完整流程示例

### 步骤1：启动后台任务

```
用户: 运行回测
我: 收到，正在后台启动回测任务...

[我执行]
exec(
    command="python /data/agents/master/run_backtest_v4.py",
    background=True
)

我: 回测已在后台启动，预计需要5分钟。你可以继续和我沟通，完成后我会通知你。
```

### 步骤2：继续沟通

```
用户: 回测框架优化得怎么样了？
我: [回答问题，不影响后台任务]
```

### 步骤3：任务完成后通知

```
[后台任务完成]
我: 
【后台任务完成通知】
📊 任务：回测（打板策略）

✅ 状态：成功
📈 结果：年化收益15.2%，夏普比率1.8

需要查看详细报告吗？
```

## 四、最佳实践

### 适合后台运行的任务

| 任务类型 | 预计耗时 | 后台运行 |
|---------|---------|---------|
| 全量数据抓取 | 2-4小时 | ✅ 推荐 |
| 回测（484天） | 5-15分钟 | ✅ 推荐 |
| 参数优化 | 10-30分钟 | ✅ 推荐 |
| 因子分析 | 5-10分钟 | ⚠️ 可选 |
| 数据验证 | 1-2分钟 | ❌ 前台 |

### 不适合后台运行的任务

- 需要实时交互的任务
- 快速查询任务（<1分钟）
- 需要用户中途确认的任务

## 五、注意事项

1. **任务ID记录**：启动后台任务后，记录任务ID以便后续查询
2. **超时处理**：长时间任务需要设置合理的yieldMs
3. **错误处理**：后台任务失败时，需要捕获并通知用户
4. **资源管理**：避免同时启动过多后台任务

## 六、对比

| 方式 | 优点 | 缺点 |
|-----|-----|-----|
| 前台运行 | 实时看到进度 | 占用会话 |
| 后台运行 | 不占用会话 | 无法实时看到进度 |
| 后台+通知 | 两全其美 | 需要额外配置 |
