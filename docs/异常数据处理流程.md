## 1. 异常数据分类

### 1.1 按严重程度分类

#### P0 级异常（致命）
**定义：** 导致策略无法运行或回测结果完全失真的数据问题

| 异常类型 | 判定标准 | 影响范围 |
|----------|----------|----------|
| 数据完全缺失 | 股票无任何历史数据 | 该股票无法回测 |
| 关键字段缺失 | 缺少 close、trade_date 等核心字段 | 所有策略无法使用 |
| 价格逻辑错误 | high < low 或 high < open/close | 回测结果失真 |
| 价格超出范围 | 价格<0 或>10000 | 策略计算错误 |

**处理时限：** 立即处理（<2 小时）

---

#### P1 级异常（严重）
**定义：** 影响部分策略或降低回测准确性的数据问题

| 异常类型 | 判定标准 | 影响范围 |
|----------|----------|----------|
| 日期连续性中断 | 缺失>5 个交易日 | 回测信号延迟 |
| 涨跌幅异常 | 涨跌幅>20%（非涨跌停） | 信号误判 |
| 成交量异常 | 成交量变化>10 倍 | 量能信号失真 |
| 涨跌停价错误 | 价格超出涨跌停限制 | 打板策略失效 |

**处理时限：** 24 小时内

---

#### P2 级异常（轻微）
**定义：** 对策略影响较小但需要关注的数据问题

| 异常类型 | 判定标准 | 影响范围 |
|----------|----------|----------|
| 少量日期缺失 | 缺失<5 个交易日 | 轻微影响 |
| 字段空值率高 | 非关键字段空值>5% | 部分因子计算受影响 |
| 疑似除权除息 | 价格跳空 15%-30% | 需人工确认 |
| 停牌标记缺失 | 停牌日未标记 | 可能产生虚假信号 |

**处理时限：** 7 天内

---

### 1.2 按异常类型分类

#### 完整性异常
- 股票列表缺失
- 交易日缺失
- 字段缺失
- 记录数不足

#### 准确性异常
- 价格错误
- 成交量错误
- 涨跌幅错误
- 涨跌停价错误

#### 一致性异常
- 开高低收关系错误
- 成交量额不匹配
- 跨表数据不一致

#### 时效性异常
- 数据更新延迟
- 除权除息未处理
- 停牌信息未同步

## 2. 异常检测流程

### 2.1 自动化检测

```
┌─────────────────────────────────────────────────────────┐
│                   数据入库前检测                          │
├─────────────────────────────────────────────────────────┤
│  1. 文件格式检查（parquet/csv）                           │
│  2. 字段存在性检查                                        │
│  3. 数据类型检查                                          │
│  4. 空值率检查                                            │
│  5. 价格范围检查                                          │
└─────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────┐
│                   数据入库后检测                          │
├─────────────────────────────────────────────────────────┤
│  1. 价格逻辑检查（high>=low, high>=open/close）           │
│  2. 涨跌幅计算检查                                        │
│  3. 涨跌停价验证                                          │
│  4. 成交量异常检测                                        │
│  5. 日期连续性检查                                        │
└─────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────┐
│                   定期全量检测                            │
├─────────────────────────────────────────────────────────┤
│  1. 股票列表完整性对比（vs Tushare 官方）                 │
│  2. 历史数据回溯校验                                      │
│  3. 跨表一致性检查                                        │
│  4. 质量评分生成                                          │
└─────────────────────────────────────────────────────────┘
```

### 2.2 检测频率

| 检测类型 | 频率 | 执行时间 |
|----------|------|----------|
| 入库前检测 | 实时 | 每次数据抓取后 |
| 入库后检测 | 实时 | 每次数据入库后 |
| 日度检测 | 每日 | 23:00 |
| 周度检测 | 每周 | 周日 02:00 |
| 月度检测 | 每月 | 1 号 02:00 |

### 2.3 检测工具


## 3. 异常处理流程

### 3.1 处理流程图

```
异常发现
   ↓
异常分级（P0/P1/P2）
   ↓
┌──────────────────┐
│   P0 级异常       │
│  - 立即告警       │
│  - 停止相关任务   │
│  - Master 介入    │
└──────────────────┘
   ↓
根因分析
   ↓
┌──────────────────────────────────────┐
│          制定修复方案                 │
├──────────────────────────────────────┤
│  方案 1：重新抓取数据                  │
│  方案 2：数据修复/插值                 │
│  方案 3：标记为不可用                  │
│  方案 4：人工介入处理                  │
└──────────────────────────────────────┘
   ↓
用户审批
   ↓
执行修复
   ↓
修复验证
   ↓
知识库归档
   ↓
流程结束
```

### 3.2 P0 级异常处理

#### 处理步骤

1. **立即告警**（<5 分钟）
   - 发送告警消息至飞书群
   - 通知 Master Agent
   - 记录异常日志

2. **停止相关任务**（<10 分钟）
   - 暂停数据更新任务
   - 暂停策略回测任务
   - 暂停选股任务

3. **根因分析**（<30 分钟）
   - 检查数据源状态
   - 检查抓取脚本日志
   - 检查网络/接口状态

4. **制定修复方案**（<1 小时）
   - 确定修复范围
   - 选择修复方法
   - 评估修复时间

5. **执行修复**（按方案）
   - 重新抓取数据
   - 或数据修复
   - 或标记不可用

6. **验证修复**（<30 分钟）
   - 运行质量检查
   - 确认问题解决
   - 恢复相关任务

#### 处理模板

```markdown
# P0 异常处理记录

## 异常信息
- 异常 ID：P0-20260312-001
- 发现时间：2026-03-12 10:30
- 异常类型：价格逻辑错误
- 影响范围：50 只股票

## 根因分析
- 直接原因：数据源接口返回异常
- 根本原因：Tushare 接口临时故障

## 处理措施
- 立即措施：暂停数据更新，标记异常数据
- 长期措施：增加数据源冗余，添加异常检测

## 修复验证
- 修复时间：2026-03-12 12:00
- 验证结果：✅ 通过
- 验证人：master-quant

## 知识库归档
- 归档位置：/knowledge-base/避坑指南/数据源异常处理.md
```

### 3.3 P1 级异常处理

#### 处理步骤

1. **记录异常**（<1 小时）
   - 记录异常详情
   - 标记异常数据
   - 通知相关人员

2. **分析原因**（<4 小时）
   - 检查数据源
   - 检查处理逻辑
   - 确定影响范围

3. **制定方案**（<8 小时）
   - 选择修复方法
   - 排期修复

4. **执行修复**（<24 小时）
   - 按方案修复
   - 记录修复过程

5. **验证关闭**（<2 小时）
   - 验证修复结果
   - 关闭异常工单

---

### 3.4 P2 级异常处理

#### 处理步骤

1. **记录异常**（<24 小时）
   - 记录到问题清单
   - 标记优先级

2. **批量处理**（<7 天）
   - 累积一定数量后批量处理
   - 或在定期维护时处理

3. **验证关闭**
   - 验证修复结果
   - 关闭问题

## 4. 修复验证流程

### 4.1 验证方法

#### 自动验证
```python
# 运行数据质量检查
checker = DataQualityChecker()
results = checker.run_full_check(ts_codes=fixed_stocks)

# 验证修复效果
if results['quality_score']['overall_score'] >= 90:
    print("✅ 修复验证通过")
else:
    print("❌ 修复验证未通过，需重新修复")
```

#### 人工验证
- 抽样检查修复数据
- 对比历史数据
- 验证策略回测结果

### 4.2 验证标准

| 验证项 | 通过标准 |
|--------|----------|
| 完整性 | 完整性得分 ≥ 95 |
| 准确性 | 准确性得分 ≥ 95 |
| 一致性 | 无逻辑错误记录 |
| 时效性 | 数据更新至最新交易日 |

## 5. 预防措施

### 5.1 数据源层面

1. **多数据源冗余**
   - 主数据源：Tushare
   - 备用数据源：AkShare
   - 应急数据源：手动导入

2. **接口监控**
   - 监控接口可用性
   - 监控接口响应时间
   - 监控接口返回质量

3. **限流保护**
   - 遵守接口调用限制
   - 实现智能重试机制
   - 避免集中调用

### 5.2 数据处理层面

1. **入库前校验**
   - 格式校验
   - 范围校验
   - 逻辑校验

2. **异常处理**
   - 捕获异常
   - 记录日志
   - 告警通知

3. **数据备份**
   - 每日备份
   - 增量备份
   - 异地备份

### 5.3 监控告警层面

1. **质量监控**
   - 实时质量评分
   - 质量趋势分析
   - 异常波动告警

2. **更新监控**

## 6. 应急预案

### 6.1 数据源故障

**场景：** Tushare 接口不可用

**应急方案：**
1. 切换至备用数据源（AkShare）
2. 如备用数据源也不可用，使用缓存数据
3. 通知用户数据延迟
4. 待数据源恢复后补全数据

### 6.2 数据质量严重下降

**场景：** 连续多只股票出现 P0 级异常

**应急方案：**
1. 立即停止数据更新
2. 回滚至上一版本数据
3. 分析原因并修复
4. 验证后恢复更新

### 6.3 系统故障

**场景：** 数据处理系统崩溃

**应急方案：**
1. 启动备用系统
2. 恢复最近备份数据
3. 排查故障原因
4. 修复后切换回主系统
