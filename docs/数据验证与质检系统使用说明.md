## 1. 系统概述

### 1.1 功能定位

数据验证与质检系统是 master-quant 量化生态的核心组件，负责：

- ✅ **数据完整性检查** - 股票列表、日期连续性、字段完整性
- ✅ **数据准确性验证** - 价格合理性、成交量/额匹配、涨跌幅计算
- ✅ **异常数据检测** - 涨停/跌停价、停牌标记、除权除息
- ✅ **数据质量报告** - 完整性评分、准确性评分、问题清单、修复建议

### 1.2 核心组件

| 组件 | 路径 | 说明 |
|------|------|------|
| 验证脚本 | `scripts/data_quality_validator.py` | 主程序 |
| 验证模块 | `modules/data_validator.py` | 核心验证逻辑 |
| 报告模板 | `templates/quality_report_template.md` | 报告模板 |
| 处理流程 | `docs/异常数据处理流程.md` | 异常处理规范 |
| 监控面板 | `dashboards/quality_monitor.md` | 质量监控 |

### 1.3 质量评分体系

```
综合得分 = 完整性得分 × 50% + 准确性得分 × 50%

完整性得分 = 股票完整性×40% + 日期连续性×30% + 字段完整性×30%
准确性得分 = 价格合理性×40% + 涨跌幅准确性×30% + 涨跌停验证×30%

评级标准：
- A (90-100 分): 优秀，可直接使用
- B (80-89 分): 良好，少量问题
- C (70-79 分): 一般，建议修复
- D (60-69 分): 较差，需要修复
- F (<60 分): 不合格，禁止使用
```

## 3. 快速开始

### 3.1 运行完整检查

```bash
# 进入工作目录
cd /home/admin/.openclaw/agents/master

# 运行完整数据质量检查（抽样 10 只股票）
python scripts/data_quality_validator.py
```

### 3.2 运行指定股票检查

```bash
# 检查单只股票
python scripts/data_quality_validator.py --stock 000001.SZ

# 检查多只股票
python scripts/data_quality_validator.py --stock 000001.SZ 600000.SH 300001.SZ
```

### 3.3 运行特定检查项

```bash
# 仅检查价格合理性
python scripts/data_quality_validator.py --check price_validity

# 仅检查日期连续性
python scripts/data_quality_validator.py --check date_continuity

# 仅检查字段完整性
python scripts/data_quality_validator.py --check field_completeness
```

### 3.4 自定义参数

```bash
# 指定抽样数量
python scripts/data_quality_validator.py --sample-size 50

# 指定输出格式
python scripts/data_quality_validator.py --output json

# 指定输出目录
python scripts/data_quality_validator.py --output-dir /path/to/reports
```

## 4. 功能详解

### 4.1 数据完整性检查

#### 4.1.1 股票列表完整性

**检查内容：**
- 对比 Tushare 官方股票列表
- 识别缺失股票
- 识别多余股票（可能已退市）

**判定标准：**
- 完整性 ≥ 99%: ✅ 优秀
- 完整性 ≥ 95%: ⚠️  良好
- 完整性 < 95%: ❌ 需修复

**示例输出：**
```
股票列表完整性:
  本地股票数：5482
  官方股票数：5489
  缺失股票数：7
  完整性：99.87% ✅
```

---

#### 4.1.2 日期连续性检查

**检查内容：**
- 对比交易日历检查缺失交易日
- 识别长期间断（>5 天）
- 计算连续性比例

**判定标准：**
- 连续性 ≥ 99%: ✅ 优秀
- 连续性 ≥ 97%: ⚠️  良好
- 连续性 < 97%: ❌ 需修复

**示例输出：**
```
日期连续性:
  实际交易日：250 天
  预期交易日：252 天
  缺失交易日：2 天
  连续性：99.21% ✅
```

---

#### 4.1.3 字段完整性检查

**检查内容：**
- 基础字段存在性（ts_code, trade_date, open, high, low, close, vol）
- 策略字段存在性（根据策略类型）
- 字段空值率

**必填字段：**
```python
基础字段：['ts_code', 'trade_date', 'open', 'high', 'low', 'close', 'vol']
打板策略：['limit', 'up_down_times', 'order_amount', 'break_limit_times']
缩量潜伏：['high', 'low', 'limit']
板块轮动：['industry', 'is_main_industry']
```

**示例输出：**
```
字段完整性:
  ts_code: 存在，空值率 0.00% ✅
  trade_date: 存在，空值率 0.00% ✅
  close: 存在，空值率 0.02% ✅
  vol: 存在，空值率 0.05% ⚠️
```

---

### 4.2 数据准确性验证

#### 4.2.1 价格合理性检查

**检查内容：**
- 价格范围（0.01 - 10000 元）
- 价格逻辑（high ≥ low, high ≥ open/close, low ≤ open/close）

**判定标准：**
- 错误率 < 0.1%: ✅ 优秀
- 错误率 < 0.5%: ⚠️  良好
- 错误率 ≥ 0.5%: ❌ 需修复

**示例输出：**
```
价格合理性:
  检查记录数：10000
  价格超出范围：0 条
  价格逻辑错误：2 条
  错误率：0.02% ✅
```

---

#### 4.2.2 成交量/额匹配检查

**检查内容：**
- 成交量与成交额变化方向一致性
- 匹配率计算

**判定标准：**
- 匹配率 ≥ 95%: ✅ 优秀
- 匹配率 ≥ 90%: ⚠️  良好
- 匹配率 < 90%: ❌ 需检查

---

#### 4.2.3 涨跌幅计算正确性

**检查内容：**
- 计算每日涨跌幅
- 识别异常涨跌幅（>20%）
- 区分市场类型（主板 10%，创业板/科创板 20%）

**判定标准：**
- 异常率 < 0.1%: ✅ 正常
- 异常率 < 1%: ⚠️  关注
- 异常率 ≥ 1%: ❌ 需核实

---

### 4.3 异常数据检测

#### 4.3.1 涨停/跌停价验证

**检查内容：**
- 计算理论涨跌停价
- 验证实际价格是否超出限制
- 分市场统计（主板/创业板/科创板）

**示例输出：**
```
涨跌停价验证:
  主板（10% 限制）: 涨停 50 次，跌停 30 次，超限 0 次 ✅
  创业板（20% 限制）: 涨停 20 次，跌停 15 次，超限 0 次 ✅
  科创板（20% 限制）: 涨停 10 次，跌停 8 次，超限 0 次 ✅
```

---

#### 4.3.2 停牌数据标记

**检查内容：**
- 识别停牌交易日
- 统计停牌天数
- 标记长期停牌期间

**停牌识别：**
- 成交量 = 0
- 或 trade_status 字段标记为'S'/'suspend'/'停牌'

---

#### 4.3.3 除权除息处理

**检查内容：**
- 识别价格跳空（15%-30%）
- 标记疑似除权除息日期
- 提示人工确认

---

### 4.4 数据质量报告

#### 4.4.1 报告内容

- 检查摘要
- 质量评分
- 完整性检查详情
- 准确性验证详情
- 异常数据清单
- 修复建议

#### 4.4.2 报告格式

- **Markdown:** 人类可读，适合查看
- **JSON:** 机器可读，适合集成
- **CSV:** 表格数据，适合分析

## 6. 常见问题

### 6.1 依赖问题

**Q: 提示 "Missing optional dependency 'pyarrow'"**

A: 安装 pyarrow 依赖：
```bash
pip install pyarrow fastparquet
```

---

**Q: 提示 "No module named 'pandas'"**

A: 激活虚拟环境并安装依赖：
```bash
source venv/bin/activate
pip install -r requirements.txt
```

---

### 6.2 数据问题

**Q: 提示 "数据加载失败"**

A: 检查以下事项：
1. 确认股票数据文件存在
2. 确认 parquet 文件未损坏
3. 确认已安装 pyarrow

---

**Q: 提示 "股票列表文件不存在"**

A: 确认 stock_basic.csv 文件位置：
```bash
ls -la data/stock_basic.csv
```

如不存在，需先运行数据抓取脚本。

---

### 6.3 性能问题

**Q: 检查速度太慢**

A: 优化建议：
1. 减少抽样数量：`--sample-size 5`
2. 仅检查特定股票：`--stock 000001.SZ`
3. 仅检查特定项目：`--check price_validity`

---

**Q: 内存占用过高**

A: 优化建议：
1. 分批检查股票
2. 使用流式处理
3. 增加系统内存

---

### 6.4 结果解读

**Q: 质量评级为 C/D/F，如何处理？**

A: 按优先级处理：
1. **F 级 (<60 分)**: 立即停止使用，全面修复
2. **D 级 (60-69 分)**: 限期修复，暂停新策略
3. **C 级 (70-79 分)**: 制定修复计划，关注关键问题

参考 `docs/异常数据处理流程.md` 执行修复。

---

**Q: 发现 P0 级异常，如何处理？**

A: 立即执行 P0 处理流程：
1. 停止相关任务
2. 通知 Master Agent
3. 根因分析
4. 制定修复方案
5. 执行修复
6. 验证关闭

详见 `docs/异常数据处理流程.md`。
