# 多智能体协作机制 - OpenCode后台运行

## 一、架构概览

```
┌─────────────────────────────────────────────────────────────┐
│                      用户 (90)                               │
│                         ↓                                   │
│              master-quant (唯一对话窗口)                      │
│                    ↙        ↘                               │
│         ┌──────────────────────────────────────┐            │
│         │           子智能体池                   │            │
│         ├──────────────────────────────────────┤            │
│         │ 执行层(5个):                          │            │
│         │  - coder (代码守护者)                 │            │
│         │  - strategy-expert (策略专家)         │            │
│         │  - test-expert (测试专家)             │            │
│         │  - doc-manager (文档管理员)           │            │
│         │  - parameter-evolver (参数进化)       │            │
│         ├──────────────────────────────────────┤            │
│         │ 支撑层(7个):                          │            │
│         │  - factor-miner (因子挖掘)            │            │
│         │  - backtest-engine (回测引擎)         │            │
│         │  - data-collector (数据采集)          │            │
│         │  - finance-learner (金融学习)         │            │
│         │  - sentiment-analyst (舆情分析)       │            │
│         │  - ops-monitor (运维监控)             │            │
│         │  - knowledge-steward (生态沉淀)       │            │
│         └──────────────────────────────────────┘            │
└─────────────────────────────────────────────────────────────┘
```

## 二、协作流程

### 流程1：任务派发（同步）

```python
# master-quant 发送任务给子智能体
sessions_send(
    sessionKey="agent:coder:main",
    message="""
【任务派发】
任务ID: TASK-20260325-001
任务类型: 代码修改
任务内容: 实现板块轮动策略
优先级: P0
截止时间: 2026-03-25 18:00

请执行并返回结果。
"""
)
```

### 流程2：后台运行（异步）

```python
# master-quant 启动后台任务
exec(
    command="/home/admin/miniconda3/envs/vnpy_env/bin/python /data/agents/master/run_backtest_v4.py",
    background=True,  # 后台运行，不阻塞会话
    yieldMs=5000      # 5秒后返回控制权
)

# 记录任务ID
task_id = "backtest-20260325-001"
```

### 流程3：状态检查

```python
# 检查后台任务状态
process(action="list")

# 查看特定任务日志
process(
    action="log",
    sessionId="任务ID"
)

# 等待任务完成（不阻塞）
process(
    action="poll",
    sessionId="任务ID",
    timeout=60000  # 最多等60秒
)
```

### 流程4：完成通知

```python
# 任务完成后，通过飞书通知用户
message(

    message="""

📊 任务：回测（打板策略）
📅 时间：2026-03-25 16:45
⏱️ 耗时：125秒
✅ 状态：成功

【核心结果】
- 年化收益：15.2%
- 夏普比率：1.8
- 最大回撤：8.5%

需要查看详细报告吗？
```

## 五、后台运行最佳实践

### 适合后台运行的任务

| 任务 | 预计耗时 | 后台运行 | 说明 |
|-----|---------|---------|-----|
| 全量数据抓取 | 2-4小时 | ✅ | yieldMs=60000 |
| 回测(484天) | 5-15分钟 | ✅ | yieldMs=30000 |
| 参数优化 | 10-30分钟 | ✅ | yieldMs=60000 |
| 因子分析 | 5-10分钟 | ✅ | yieldMs=30000 |
| 代码开发 | 10-30分钟 | ✅ | yieldMs=30000 |
| 数据验证 | 1-2分钟 | ❌ | 前台运行 |

### 超时处理

```python
# 方案1: 设置合理的yieldMs
exec(
    command="python run_backtest.py",
    background=True,
    yieldMs=60000  # 1分钟后返回
)

# 方案2: 分阶段运行
# 第一阶段: 数据准备
exec(command="python prepare_data.py", background=True)
# 第二阶段: 回测运行
exec(command="python run_backtest.py", background=True)

# 方案3: 直接SSH运行（最长时间任务）
exec(command="nohup python run_backtest.py > log.txt 2>&1 &", background=True)
```

## 六、注意事项

### ⚠️ 禁止行为

1. **禁止阻塞会话**: 长时间任务必须后台运行
2. **禁止忘记通知**: 任务完成后必须通知用户
3. **禁止丢失任务**: 必须记录任务ID和状态

### ✅ 强制行为

1. **派发任务时**: 明确任务ID、内容、截止时间
2. **后台运行时**: 设置合理的yieldMs
3. **完成时**: 通知用户结果
