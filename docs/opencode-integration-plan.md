## 二、工具使用规则调整

### 2.1 禁用 → 启用（需用户确认）

| 工具 | 原规则 | 新建议 | 理由 |
|------|--------|--------|------|
| `read` | 禁止直接使用 | ✅ 启用 | OpenCode 提供安全的文件读取，master 需要直接查看代码 |
| `write` | 禁止直接使用 | ⚠️ 需确认后使用 | 代码修改仍需用户确认，但可先写入草稿 |
| `web_search` | 禁止直接使用 | ❌ 保持禁用 | 通过 data-collector 统一管理 |
| `trading_api` | 禁止 | ❌ 永久禁用 | 红线规则 |

### 2.2 新增能力使用

| 工具 | 用途 | 示例 |
|------|------|------|
| `sessions_spawn` | 动态创建子智能体 | `sessions_spawn(task="优化策略参数", agentId="parameter-evolver")` |
| `subagents` | 管理运行中的子智能体 | `subagents(action="list")` 查看所有运行中的子智能体 |
| `subagents(action="kill")` | 中止失控的子智能体 | 终止卡住的任务 |

## 三、调度方式优化

### 3.1 当前方式（静态配置）
```
问题：
1. 14个子智能体长期运行，消耗资源
2. 大部分显示 abortedLastRun: true
3. 无法动态调整智能体数量
```

### 3.2 建议方式（动态创建）

**方案A：混合模式（推荐）**
```
核心智能体（常驻）：
- coder: 代码守护者
- strategy-expert: 策略专家
- test-expert: 测试专家

按需创建（sessions_spawn）：
- parameter-evolver: 参数优化任务
- backtest-engine: 回测任务
- factor-miner: 因子挖掘任务
- data-collector: 数据采集任务
- sentiment-analyst: 舆情分析任务
- ops-monitor: 监控任务
- knowledge-steward: 知识库沉淀任务
- doc-manager: 文档管理任务
- finance-learner: 学习任务
```

**方案B：完全动态**
```
所有子智能体按需创建，任务完成后自动清理
优点：资源高效
缺点：启动延迟
```

## 四、工作流程重构

### 4.1 标准流程（保留10步SOP）

```
步骤1: 指令接收与双向校验
       ↓ [启用 read 直接查看代码]
步骤2: 任务拆解与精准分发
       ↓ [使用 sessions_spawn 按需创建子智能体]
步骤3: 执行过程管控与支撑中转
       ↓ [使用 subagents 监控运行状态]
步骤4: 执行结果全量校验
       ↓ [使用 sessions_history 获取结果]
步骤5: 结果汇总与用户交付
       ↓ [使用 message 发送到飞书]
步骤6: 用户审批与流程分支
       ↓
步骤7: 代码版本更新与全链路复盘
       ↓ [使用 write 写入用户确认后的代码]
步骤8: 知识库全量沉淀
       ↓ [使用 memory_store 沉淀经验]
步骤9: 全生态能力迭代
       ↓
步骤10: 待命状态
```

### 4.2 异常处理流程（新增）

```
发现子智能体卡住：
1. subagents(action="list") 查看运行状态
2. subagents(action="kill", target="xxx") 终止问题智能体
3. sessions_spawn 重新创建智能体继续任务
4. 记录错误到避坑指南
```

## 五、记忆系统整合

### 5.1 OpenCode 记忆工具 vs 六层记忆系统

| OpenCode 工具 | 对应层级 | 说明 |
|--------------|---------|------|
| `memory_store` | Layer 0/1 | 存储 Core Memory 和 Knowledge |
| `memory_recall` | 所有层级 | 混合检索 |
| `memory_update` | Layer 3 | 更新工作记忆 |
| `memory_forget` | 所有层级 | 清理过期记忆 |

### 5.2 整合方案

**保留六层记忆架构**，但使用 OpenCode 的 memory_* 工具实现：

```python
# 原来（memory_manager.py 脚本）
python memory/memory_manager.py --action add_error --content "错误信息"

# 现在（OpenCode 工具）
memory_store(
    text="错误信息",
    category="error",
    scope="master-quant"
)
```
