## 1. 全量抓取模式（抓取 + 回测）

**模式标识：** `--mode "抓取 + 回测"`

### 功能验证
- [ ] **能正常启动**
  - 执行命令：`python main.py --mode "抓取 + 回测" --dry-run`
  - 预期：程序启动，无报错，显示帮助信息或配置摘要
  
- [ ] **能读取配置的时间范围**
  - 执行命令：`python main.py --mode "抓取 + 回测" --start-date 2020-01-01 --end-date 2026-03-12 --dry-run`
  - 预期：日志中显示正确的时间范围配置
  
- [ ] **能正常抓取数据**
  - 执行命令：`python main.py --mode "抓取 + 回测" --start-date 2026-03-10 --end-date 2026-03-12`
  - 预期：
    - 日志显示数据抓取进度
    - 数据文件生成（`data/*.parquet` 或 `data/*.csv`）
    - 无 API 请求错误
    
- [ ] **能保存进度**
  - 检查点：`data/checkpoint.json` 或 `data/progress.json`
  - 预期：进度文件存在，包含已抓取日期列表

### 性能验证
- [ ] 抓取速度 ≥ 100 条/秒
- [ ] 内存占用 < 2GB
- [ ] 无内存泄漏（连续运行 10 分钟内存稳定）

### 异常处理
- [ ] 网络中断后能恢复
- [ ] API 限流时能自动重试
- [ ] 数据格式错误时能跳过并记录
