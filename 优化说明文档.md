# 数据抓取代码优化说明文档

## 一、优化背景

原始代码 `Untitled-12_comprehensive_optimized.py` 存在以下问题：
1. 并发线程数过高（20），对测试环境压力过大
2. 限流配置过激（4000 次/分钟），可能触发 Tushare 限流
3. 数据存储全部使用 CSV，占用空间大、读取效率低
4. 存储路径分散，不便于统一管理
5. 日志输出缺少抓取进度和压缩比等关键信息

## 二、优化目标

### 2.1 性能优化
- 降低并发线程数至 4，适配测试环境
- 调整限流至 500 次/分钟，符合服务器配置
- 引入 Parquet+Snappy 压缩，减少存储空间
- 使用 SQLite 存储结构化数据，提升查询效率

### 2.2 路径优化
- 统一数据存储路径：`/home/admin/.openclaw/agents/master/quant_data_project/data/`
- 统一日志输出路径：`/home/admin/.openclaw/agents/master/quant_data_project/logs/`

### 2.3 日志优化
- 增加抓取进度日志（每 100 只股票输出一次）
- 增加压缩比日志（原始大小、压缩后大小、压缩比）

## 三、优化详解

### 3.1 并发线程数调整

**优化前：** `max_workers = 20`  
**优化后：** `max_workers = 4`

**原因：**
- 测试环境不需要高并发
- 降低服务器负载
- 便于调试和问题排查

**影响：**
- 抓取速度降低约 80%
- 服务器 CPU/内存占用降低约 75%
- 网络请求更平稳

### 3.2 限流配置调整

**优化前：** `max_requests_per_minute = 4000`  
**优化后：** `max_requests_per_minute = 500`

**原因：**
- 根据服务器实际配置调整
- 避免触发 Tushare API 限流
- 预留充足缓冲空间

**影响：**
- 请求速率降低 87.5%
- 大幅降低被封禁风险
- 抓取时间相应延长

### 3.3 数据压缩优化

#### 3.3.1 日线行情 - Parquet + Snappy

**优化前：**
```python
df_daily.to_csv(filepath, index=False, encoding='utf-8-sig')
```

**优化后：**
```python
utils.save_to_parquet_snappy(df_daily, filepath.replace('.csv', '.parquet'), data_type="日线行情")
```

**优势：**
- 压缩比：3-5 倍
- 读取速度：提升 3-5 倍（列式存储）
- 支持字段级读取，无需加载整列

**示例压缩比日志：**
```
📊 压缩比日志 | 日线行情：原始 15.23MB → 压缩后 3.45MB | 压缩比：4.41x
```

#### 3.3.2 财务数据 - SQLite

**优化前：**
```python
df_fina.to_csv(filepath, index=False, encoding='utf-8-sig')
```

**优化后：**
```python
utils.save_to_sqlite(df_fina, db_path, table_name='fina_indicator', data_type="财务数据")
```

**优势：**
- 支持 SQL 查询，灵活性高
- 支持索引，查询速度快
- 支持事务，数据一致性好
- 多表关联查询高效

**示例压缩比日志：**
```
📊 压缩比日志 | 财务数据：数据库大小 8.76MB
```

#### 3.3.3 资金流向 - Parquet + Snappy

同日线行情，使用 Parquet 压缩存储。

#### 3.3.4 龙虎榜 - SQLite

同财务数据，使用 SQLite 存储，便于按日期、股票代码查询。

### 5.2 数据迁移（可选）

如需将原有 CSV 数据转换为新格式：

```python
import pandas as pd
import pyarrow as pa
import pyarrow.parquet as pq

# CSV 转 Parquet
df = pd.read_csv('old_data.csv')
table = pa.Table.from_pandas(df)
pq.write_table(table, 'new_data.parquet', compression='snappy')

# CSV 转 SQLite
import sqlite3
conn = sqlite3.connect('new_data.db')
df.to_sql('table_name', conn, if_exists='replace', index=False)
conn.close()
```

### 5.3 读取新格式数据


```python

import pandas as pd

df = pd.read_sql_query('SELECT * FROM table_name', conn)

```

## 七、注意事项

### 7.1 兼容性

- **降级兼容：** 如果 pyarrow 不可用，自动降级为 CSV 存储
- **原有数据：** 原有 CSV 数据可继续使用，无需迁移
- **API 接口：** Flask API 读取逻辑保持不变

### 7.2 性能调优

根据实际运行情况，可调整以下参数：

```python
FETCH_OPTIMIZATION = {
    'max_workers': 4,              # 根据服务器 CPU 核心数调整
    'max_requests_per_minute': 500  # 根据 Tushare 积分调整
}
```

### 7.3 故障排查

**问题 1：Parquet 保存失败**
- 检查 pyarrow 是否安装：`pip install pyarrow`
- 检查磁盘空间是否充足
- 查看日志中的详细错误信息

**问题 2：SQLite 锁定**
- 检查是否有多个进程同时写入
- 确保写入完成后调用 `conn.close()`
- 考虑使用 WAL 模式提升并发性能

**问题 3：抓取速度慢**
- 适当增加 `max_workers`（如 8、12）
- 适当增加 `max_requests_per_minute`（如 1000、2000）
- 检查网络连接质量
