## 一、战法研究

### 1.1 打板战法体系

#### 1.1.1 龙头战法

**核心逻辑：**
- 追逐市场最强龙头股，利用龙头溢价和情绪惯性获利
- 龙头股特征：板块领涨、连板高度最高、成交量最大、辨识度最强

**入场条件：**
- 确认龙头地位（3 板以上，板块内最先涨停）
- 分歧转一致时刻（开板回封、弱转强）
- 5 日线/10 日线支撑位低吸

**出场条件：**
- 龙头见顶信号（高位放量滞涨、跌停）
- 板块整体退潮
- 跌破关键均线（5 日/10 日线）

**统计数据（基于历史回测）：**
| 指标 | 数值 |
|------|------|
| 胜率 | 45%-55% |
| 盈亏比 | 2.5:1 - 4:1 |
| 平均持仓周期 | 3-7 天 |
| 最大回撤 | 15%-25% |

**适用场景：**
- ✅ 牛市主升浪
- ✅ 震荡市热点轮动期
- ❌ 熊市主跌浪
- ❌ 市场成交量<6000 亿

---

#### 1.1.2 首板战法

**核心逻辑：**
- 捕捉首个涨停板，博弈连板溢价
- 重点：题材新颖度、封板质量、时间优先级

**入场条件：**
- 早盘 10:30 前涨停（时间优先）
- 封单金额>5000 万
- 题材为当日热点/新题材
- 成交量适中（换手率 5%-15%）

**出场条件：**
- 次日不连板即出
- 低开低走止损
- 连板失败冲高回落

**统计数据：**
| 指标 | 数值 |
|------|------|
| 胜率 | 35%-45% |
| 盈亏比 | 1.5:1 - 2.5:1 |
| 平均持仓周期 | 1-2 天 |
| 连板率 | 25%-35% |

**适用场景：**
- ✅ 震荡市
- ✅ 题材爆发初期
- ❌ 市场情绪冰点期
- ❌ 监管高压期

---

#### 1.1.3 连板战法

**核心逻辑：**
- 参与 2 板及以上连板股，博弈情绪加速
- 核心：辨识度、板块效应、情绪周期

**入场条件：**
- 2-3 板分歧转一致
- 板块有梯队（首板 + 连板）
- 市场情绪上升期

**出场条件：**
- 断板即出
- 高位放量滞涨
- 情绪周期拐点

**统计数据：**
| 指标 | 数值 |
|------|------|
| 胜率 | 40%-50% |
| 盈亏比 | 2:1 - 3:1 |
| 平均持仓周期 | 2-4 天 |
| 3 板以上晋级率 | 30%-40% |

### 1.2 潜伏战法体系

#### 1.2.1 低吸战法

**核心逻辑：**
- 在强势股回调时低吸，博弈反弹/二波
- 关键：识别真回调 vs 真下跌

**入场条件：**
- 前期龙头/强势股
- 回调至关键支撑（10 日/20 日线、前期平台）
- 缩量企稳信号
- 板块逻辑未破坏

**出场条件：**
- 反弹至前高附近
- 跌破支撑位止损
- 时间止损（5 天不反弹）

**统计数据：**
| 指标 | 数值 |
|------|------|
| 胜率 | 50%-60% |
| 盈亏比 | 2:1 - 3:1 |
| 平均持仓周期 | 3-5 天 |

---

#### 1.2.2 埋伏战法

**核心逻辑：**
- 提前布局预期事件/政策驱动
- 核心：信息差、预期差、时间窗口

**入场条件：**
- 明确催化剂（政策/事件/财报）
- 提前 1-2 周布局
- 低位启动信号

**出场条件：**
- 利好兑现即出
- 预期落空止损
- 时间窗口结束

**统计数据：**
| 指标 | 数值 |
|------|------|
| 胜率 | 45%-55% |
| 盈亏比 | 2:1 - 4:1 |
| 平均持仓周期 | 5-15 天 |

### 1.3 轮动战法体系

**识别信号：**
- 主线板块高位滞涨

- 龙头股切换

**操作策略：**
- 卖出高位板块龙头

## 二、市场环境分析

### 2.1 牛熊震荡识别模型

#### 2.1.1 多维度识别指标

**1. 均线系统**
```
牛市信号：
- 上证指数 > 250 日均线
- 250 日均线向上
- 股价 > 60 日 > 120 日 > 250 日均线（多头排列）

熊市信号：
- 上证指数 < 250 日均线
- 250 日均线向下
- 股价 < 60 日 < 120 日 < 250 日均线（空头排列）

震荡市信号：
- 股价围绕 250 日均线波动
- 均线纠缠
- 波动率<20%
```

**2. 市场宽度指标**
```
牛市：
- 创新高股票数 > 创新低股票数 (3:1 以上)
- 上涨家数/下跌家数 > 2
- 涨停家数 > 50 家

熊市：
- 创新低股票数 > 创新高股票数
- 上涨家数/下跌家数 < 0.5
- 跌停家数 > 20 家

震荡市：
- 比值在 0.8-1.2 之间
- 涨停/跌停家数相当
```

**3. 成交量指标**
```
牛市：
- 两市成交额 > 10000 亿
- 量价齐升

熊市：
- 两市成交额 < 6000 亿
- 缩量下跌

震荡市：
- 成交额 6000-10000 亿
- 成交量波动
```

**4. 情绪指标**
```
牛市：
- 涨停板封板率 > 70%
- 连板高度 > 7 板
- 昨日涨停今日溢价 > 3%

熊市：
- 封板率 < 40%
- 连板高度 < 3 板
- 昨日涨停今日溢价 < 0%

震荡市：
- 封板率 40%-70%
- 连板高度 3-5 板
```

#### 2.1.2 综合评分模型

```python
def market_regime_score():
    """
    市场环境综合评分 (0-100)
    >70: 牛市
    30-70: 震荡市
    <30: 熊市
    """
    score = 0
    
    # 均线系统 (25 分)
    if price > ma250 and ma250_slope > 0:
        score += 25
    elif price < ma250 and ma250_slope < 0:
        score += 0
    else:
        score += 12
    
    # 市场宽度 (25 分)
    width_ratio = new_highs / new_lows
    if width_ratio > 3:
        score += 25
    elif width_ratio < 0.5:
        score += 0
    else:
        score += 12 * width_ratio
    
    # 成交量 (25 分)
    if volume > 1000 亿：
        score += 25
    elif volume < 6000 亿：
        score += 0
    else:
        score += 12.5 + (volume - 6000) / 4000 * 12.5
    
    # 情绪指标 (25 分)
    if limit_up_rate > 0.7 and连板高度 > 7:
        score += 25
    elif limit_up_rate < 0.4 and连板高度 < 3:
        score += 0
    else:
        score += 12
    
    return score
```

## 三、外围因素分析

### 3.1 战争/地缘政治影响

**影响路径：**
```
地缘冲突 → 风险偏好下降 → 资金避险 → 股市下跌
         → 大宗商品上涨 → 通胀预期 → 货币政策收紧
         → 供应链中断 → 相关行业受损
```

**监控指标：**
- VIX 恐慌指数
- 黄金价格
- 原油价格
- 美元指数
- 国债收益率

**历史规律：**
- 短期冲击（1-2 周）：市场普跌 5%-15%
- 中期影响（1-3 月）：结构性分化
- 长期影响：取决于冲突规模和持续时间

**应对策略：**
- 冲突初期：降低仓位，配置避险资产
- 冲突中期：关注军工、能源、农业板块
- 冲突后期：布局受损行业修复

### 3.3 利好利空/预期兑现逻辑

**核心规律：**
```
预期阶段：股价提前反应（涨/跌）
兑现阶段：利好出尽是利空，利空出尽是利好
```

**操作策略：**
- 预期阶段：提前布局，享受预期溢价
- 兑现阶段：提前离场，避免利好出尽
- 超预期：继续持有
- 低于预期：立即止损

**典型案例：**
- 财报季：提前 1-2 周布局，财报前离场
- 政策会议：会议前布局，会议后离场
- 产品发布：发布前布局，发布后离场

### 4.3 回测陷阱避免

#### 4.3.1 未来函数（Look-ahead Bias）

**常见错误：**
```python
# ❌ 错误：使用当日收盘价决定当日交易
if close > ma20:  # close 是当日收盘价
    buy()  # 但交易发生在盘中

# ✅ 正确：使用昨日数据决定今日交易
if close[1] > ma20[1]:  # 使用昨日数据
    buy()  # 今日开盘交易
```

**避免方法：**
- 严格使用 T-1 数据决定 T 日交易
- 盘中数据只能用开盘至当前时刻
- 财务数据用公告日期而非报告期

---

#### 4.3.2 过拟合（Overfitting）

**识别信号：**
- 样本内表现极好，样本外极差
- 参数过多（>10 个自由参数）
- 策略逻辑过于复杂

**避免方法：**
1. **参数简化**：减少自由参数至 3-5 个
2. **样本外验证**：保留 20%-30% 数据做 OOS
3. **交叉验证**：多时间段验证
4. **逻辑优先**：策略逻辑先于参数优化
5. **正则化**：对复杂策略惩罚

```python
# 过拟合检测
def check_overfitting(in_sampleSharpe, out_sampleSharpe):
    ratio = out_sampleSharpe / in_sampleSharpe
    if ratio < 0.5:
        return "严重过拟合"
    elif ratio < 0.7:
        return "可能过拟合"
    else:
        return "通过"
```

---

#### 4.3.3 幸存者偏差（Survivorship Bias）

**问题：**
- 只回测当前存活股票
- 忽略退市/ST 股票
- 导致结果过于乐观

**解决方案：**
- 使用全量历史数据（包含退市股票）
- 使用聚宽/米筐等提供退市数据的服务
- 手动整理退市股票列表

---

#### 4.3.4 其他陷阱

**流动性陷阱：**
- 假设任意数量都能成交
- 忽略涨跌停无法交易

**解决：**
- 设置成交量限制（<20% 日成交量）
- 涨跌停不交易

**复权陷阱：**
- 前复权导致历史价格为负
- 后复权导致信号漂移

**解决：**
- 使用不复权价格 + 手动处理分红

### 4.4 滑点/手续费/冲击成本建模

#### 4.4.1 手续费模型

```python
# A 股手续费标准
def calculate_fee(amount, side):
    """
    amount: 交易金额
    side: 'buy' or 'sell'
    """
    # 佣金（万 2.5，最低 5 元）
    commission = max(5, amount * 0.00025)
    
    # 印花税（仅卖出，万 5）
    stamp_tax = amount * 0.0005 if side == 'sell' else 0
    
    # 过户费（万 0.2）
    transfer_fee = amount * 0.00002
    
    return commission + stamp_tax + transfer_fee
```

---

#### 4.4.2 滑点模型

**固定滑点：**
```python
slippage = 0.001  # 0.1%
exec_price = signal_price * (1 + slippage)  # 买入
exec_price = signal_price * (1 - slippage)  # 卖出
```

**动态滑点（基于成交量）：**
```python
def dynamic_slippage(order_size, daily_volume):
    """
    基于订单大小占成交量比例的滑点
    """
    volume_ratio = order_size / daily_volume
    base_slippage = 0.001
    impact_factor = 0.01  # 每 1% 成交量增加 0.01% 滑点
    
    return base_slippage + volume_ratio * impact_factor
```

---

#### 4.4.3 冲击成本模型

**线性模型：**
```python
def market_impact(order_size, daily_volume, volatility):
    """
    冲击成本 = 基础冲击 + 波动率调整
    """
    volume_ratio = order_size / daily_volume
    base_impact = 0.001 * volume_ratio
    vol_adjustment = 0.0005 * volatility * volume_ratio
    
    return base_impact + vol_adjustment
```

**平方根模型（更贴近实际）：**
```python
def market_impact_sqrt(order_size, daily_volume):
    """
    冲击成本与订单大小的平方根成正比
    """
    volume_ratio = order_size / daily_volume
    impact_coefficient = 0.01
    
    return impact_coefficient * np.sqrt(volume_ratio)
```

#### 4.4.4 综合成本模型

```python
class TradingCostModel:
    def __init__(self):
        self.commission_rate = 0.00025
        self.stamp_tax_rate = 0.0005
        self.transfer_fee_rate = 0.00002
        self.base_slippage = 0.001
        self.impact_coefficient = 0.01
    
    def calculate_total_cost(self, price, shares, side, daily_volume, volatility):
        amount = price * shares
        
        # 手续费
        fee = self.calculate_fee(amount, side)
        
        # 滑点
        slippage = self.base_slippage + 0.01 * (shares / daily_volume)
        
        # 冲击成本
        impact = self.impact_coefficient * np.sqrt(shares / daily_volume)
        
        # 总成本
        if side == 'buy':
            total_cost = fee + amount * (slippage + impact)
        else:
            total_cost = fee + amount * (slippage + impact) + amount * self.stamp_tax_rate
        
        return total_cost
```

### 5.2 市场环境识别模型

```python
class MarketRegimeModel:
    """市场环境识别模型"""
    
    def __init__(self):
        self.weights = {
            'ma_system': 0.25,
            'market_breadth': 0.25,
            'volume': 0.25,
            'sentiment': 0.25
        }
    
    def calculate_score(self, market_data):
        """
        计算市场环境评分 (0-100)
        
        Parameters:
        market_data: dict containing:
            - price, ma250, ma250_slope
            - new_highs, new_lows
            - volume (亿)
            - limit_up_rate, limit_up_height
        """
        score = 0
        
        # 均线系统评分
        if market_data['price'] > market_data['ma250']:
            score += 25 if market_data['ma250_slope'] > 0 else 12
        else:
            score += 0 if market_data['ma250_slope'] < 0 else 12
        
        # 市场宽度评分
        width_ratio = market_data['new_highs'] / max(1, market_data['new_lows'])
        if width_ratio > 3:
            score += 25
        elif width_ratio < 0.5:
            score += 0
        else:
            score += 12 * width_ratio
        
        # 成交量评分
        volume = market_data['volume']
        if volume > 10000:
            score += 25
        elif volume < 6000:
            score += 0
        else:
            score += 12.5 + (volume - 6000) / 4000 * 12.5
        
        # 情绪评分
        if market_data['limit_up_rate'] > 0.7 and market_data['limit_up_height'] > 7:
            score += 25
        elif market_data['limit_up_rate'] < 0.4 and market_data['limit_up_height'] < 3:
            score += 0
        else:
            score += 12
        
        return score
    
    def get_regime(self, score):
        """根据评分返回市场环境"""
        if score > 70:
            return '牛市'
        elif score > 30:
            return '震荡市'
        else:
            return '熊市'
    
    def get_recommended_strategy(self, regime):
        """根据市场环境推荐策略"""
        recommendations = {
            '牛市': ['龙头战法', '连板战法'],
            '震荡市': ['首板战法', '高低切换', '低吸战法'],
            '熊市': ['埋伏战法', '空仓观望']
        }
        return recommendations.get(regime, [])
```
