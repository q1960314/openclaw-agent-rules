### 3.2 核心组件说明

#### 3.2.1 插件基类 `PluginBase`

**文件：** `plugins/plugin_base.py`

**核心功能：**
- ✅ 定义插件生命周期（加载→初始化→激活→停用→卸载）
- ✅ 管理插件状态（8 种状态：UNLOADED/LOADING/LOADED/INITIALIZING/ACTIVE/INACTIVE/ERROR/UNLOADING/STOPPED）
- ✅ 提供错误处理框架
- ✅ 记录插件统计信息（加载时间、初始化时间、错误信息等）

**关键方法：**
```python
# 抽象方法（必须实现）
on_load() -> bool           # 加载回调
on_init() -> bool           # 初始化回调
on_activate() -> bool       # 激活回调
on_deactivate() -> bool     # 停用回调
on_unload() -> bool         # 卸载回调

# 生命周期执行方法
load() -> bool              # 执行加载流程
initialize() -> bool        # 执行初始化流程
activate() -> bool          # 执行激活流程
deactivate() -> bool        # 执行停用流程
unload() -> bool            # 执行卸载流程

# 工具方法
get_state() -> PluginState  # 获取状态
is_active() -> bool         # 检查是否活跃
get_stats() -> Dict         # 获取统计信息
```

**【优化】标记点：**
- 插件状态枚举 `PluginState`
- 插件元信息类 `PluginInfo`
- 完整的生命周期管理
- 统一的错误处理框架

### 4.2 兼容性说明

#### 4.2.1 DataSourcePlugin vs DataSource

| 特性 | DataSource (modules) | DataSourcePlugin (plugins) |
|------|---------------------|---------------------------|
| 定位 | 轻量级接口 | 完整插件化 |
| 生命周期 | 简单 | 完整（加载/初始化/激活/停用/卸载） |
| 热插拔 | ❌ | ✅ |
| 依赖管理 | ❌ | ✅ |
| 状态监控 | 基础 | 完整统计 |
| 推荐场景 | 简单数据源 | 需要热插拔的数据源 |

**兼容性：** 两者方法签名完全一致，可以互换使用

#### 4.2.2 迁移路径

```
现有代码 (modules/)
    │
    ├── 保持不变（向后兼容）
    │
    └── 新增功能 → 使用插件架构 (plugins/)
```

## 五、使用示例

### 5.1 加载和使用插件

```python
from plugins import PluginManager

# 1. 获取管理器单例
manager = PluginManager.get_instance()

# 2. 加载所有插件
count = manager.load_all_plugins()
print(f"加载了 {count} 个插件")

# 3. 激活所有插件
manager.activate_all_plugins()

# 4. 获取策略插件
strategy = manager.get_plugin('limit_up_strategy')

# 5. 生成交易信号
market_data = {
    'daily': daily_df,
    'top_list': top_list_df,
}
signals = strategy.generate_signals(market_data, {})

# 6. 处理信号
for signal in signals:
    print(f"{signal.signal_type}: {signal.ts_code} @ {signal.price}")

# 7. 健康检查
health = manager.health_check()
print(f"健康状态：{'健康' if health['healthy'] else '异常'}")
```
