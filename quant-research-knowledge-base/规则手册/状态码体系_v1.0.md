# 状态码体系 - StatusCode System v1.0

**用途**: 智能体协作的状态驱动机制
**更新时间**: 2026-03-18

---

## 一、状态码分类

### 1.1 成功状态 (1xx)

| 状态码 | 名称 | 说明 |
|--------|------|------|
| STATUS_SUCCESS | 成功 | 任务执行成功 |
| STATUS_DATA_OK | 数据正常 | 数据采集/验证成功 |
| STATUS_STRATEGY_VALIDATED | 策略验证通过 | 策略回测通过 |
| STATUS_BACKTEST_PASS | 回测通过 | 回测结果达标 |
| STATUS_CODE_DEPLOYED | 代码部署成功 | 代码更新完成 |
| STATUS_DOC_UPDATED | 文档更新成功 | 知识库更新完成 |

### 1.2 数据状态 (2xx)

| 状态码 | 名称 | 说明 | 触发动作 |
|--------|------|------|----------|
| STATUS_DATA_ERROR | 数据错误 | 数据采集失败 | → ops-monitor |
| STATUS_DATA_STALE | 数据过时 | 数据超过有效期 | → data-collector |
| STATUS_DATA_INCOMPLETE | 数据不完整 | 缺少必要字段 | → data-collector |

### 1.3 策略状态 (3xx)

| 状态码 | 名称 | 说明 | 触发动作 |
|--------|------|------|----------|
| STATUS_STRATEGY_NEW | 新策略 | 需要验证 | → backtest-engine |
| STATUS_STRATEGY_REJECTED | 策略拒绝 | 未达标准 | → strategy-expert 修改 |
| STATUS_STRATEGY_OPTIMIZED | 策略优化 | 参数优化完成 | → test-expert |

### 1.4 回测状态 (4xx)

| 状态码 | 名称 | 说明 | 触发动作 |
|--------|------|------|----------|
| STATUS_BACKTEST_RUNNING | 回测中 | 正在执行 | 等待 |
| STATUS_BACKTEST_PASS | 回测通过 | 达到目标 | → parameter-evolver |
| STATUS_BACKTEST_FAIL | 回测失败 | 未达标准 | → strategy-expert |

### 1.5 代码状态 (5xx)

| 状态码 | 名称 | 说明 | 触发动作 |
|--------|------|------|----------|
| STATUS_CODE_CHANGED | 代码变更 | 需要测试 | → test-expert |
| STATUS_CODE_TESTED | 代码测试通过 | 可部署 | → doc-manager |
| STATUS_CODE_ERROR | 代码错误 | 执行失败 | → coder 修复 |

### 1.6 风险状态 (6xx)

| 状态码 | 名称 | 说明 | 触发动作 |
|--------|------|------|----------|
| STATUS_RISK_LOW | 低风险 | 正常监控 | 记录 |
| STATUS_RISK_MEDIUM | 中风险 | 需关注 | 告警 |
| STATUS_RISK_HIGH | 高风险 | 需处理 | → master-quant |
| STATUS_RISK_CRITICAL | 严重风险 | 紧急处理 | 自动预案 + master-quant |

### 1.7 流程状态 (7xx)

| 状态码 | 名称 | 说明 | 触发动作 |
|--------|------|------|----------|
| STATUS_PROCESS_SLOW | 流程缓慢 | 超时风险 | → 降级处理 |
| STATUS_PROCESS_TIMEOUT | 流程超时 | 执行失败 | → 重试/告警 |
| STATUS_PROCESS_ERROR | 流程错误 | 执行异常 | → ops-monitor |

## 二、状态流转规则

### 2.1 代码修改流程

```
STATUS_CODE_CHANGED (coder)
    ↓
STATUS_CODE_TESTED (test-expert)
    ↓
STATUS_CODE_DEPLOYED (doc-manager)
    ↓
STATUS_DOC_UPDATED (knowledge-steward)
    ↓
STATUS_SUCCESS
```

### 2.2 策略验证流程

```
STATUS_STRATEGY_NEW (strategy-expert)
    ↓
STATUS_BACKTEST_RUNNING (backtest-engine)
    ↓
STATUS_BACKTEST_PASS / STATUS_BACKTEST_FAIL
    ↓ (通过)
STATUS_STRATEGY_OPTIMIZED (parameter-evolver)
    ↓
STATUS_STRATEGY_VALIDATED
```

### 2.3 风险处理流程

```
STATUS_RISK_HIGH (ops-monitor 检测)
    ↓
→ master-quant 告警
    ↓
用户确认处理方案
    ↓
执行处理
    ↓
STATUS_SUCCESS
```

## 三、状态码使用示例

```python
class StatusCode:
    """状态码定义"""
    
    # 成功
    SUCCESS = "STATUS_SUCCESS"
    
    # 数据
    DATA_OK = "STATUS_DATA_OK"
    DATA_ERROR = "STATUS_DATA_ERROR"
    
    # 策略
    STRATEGY_NEW = "STATUS_STRATEGY_NEW"
    STRATEGY_VALIDATED = "STATUS_STRATEGY_VALIDATED"
    
    # 回测
    BACKTEST_RUNNING = "STATUS_BACKTEST_RUNNING"
    BACKTEST_PASS = "STATUS_BACKTEST_PASS"
    BACKTEST_FAIL = "STATUS_BACKTEST_FAIL"
    
    # 代码
    CODE_CHANGED = "STATUS_CODE_CHANGED"
    CODE_TESTED = "STATUS_CODE_TESTED"
    
    # 风险
    RISK_LOW = "STATUS_RISK_LOW"
    RISK_MEDIUM = "STATUS_RISK_MEDIUM"
    RISK_HIGH = "STATUS_RISK_HIGH"
    RISK_CRITICAL = "STATUS_RISK_CRITICAL"


class StatusHandler:
    """状态处理器"""
    
    ROUTES = {
        StatusCode.CODE_CHANGED: ["test-expert"],
        StatusCode.BACKTEST_PASS: ["parameter-evolver"],
        StatusCode.RISK_HIGH: ["master-quant"],
    }
    
    def handle_status(self, status: str, context: dict):
        """根据状态码路由到对应智能体"""
        handlers = self.ROUTES.get(status, [])
        return handlers
```
