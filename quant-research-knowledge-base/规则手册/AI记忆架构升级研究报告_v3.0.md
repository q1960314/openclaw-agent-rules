### 2.4 OpenCode Working Memory 架构模式

```
┌─────────────────────────────────────────────────────────────┐
│               OpenCode 四层工作记忆架构                        │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  Layer 1: Core Memory (核心记忆)                             │
│  ├─ goal (1000 chars) - 当前目标                             │
│  ├─ progress (2000 chars) - 进度状态                         │
│  └─ context (1500 chars) - 关键上下文                        │
│                                                              │
│  Layer 2: Working Memory (工作记忆)                          │
│  ├─ Slots (FIFO): errors, decisions, todos, dependencies    │
│  └─ Pool (Ranked): file-paths, recent, mentions, decay      │
│                                                              │
│  Layer 3: Memory Pressure Monitor (压力监控)                 │
│  ├─ Token 实时追踪                                           │
│  ├─ 75% moderate → 90% high 告警                            │
│  └─ 自动干预机制                                             │
│                                                              │
│  Layer 4: Storage Governance (存储治理)                       │
│  ├─ 最多 300 文件/会话                                       │
│  ├─ 7 天 TTL 自动清理                                        │
│  └─ 后台静默运行                                             │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

**关键设计理念：**
1. **Slot + Pool 双层工作记忆** - 保证关键信息可见 + 按相关性排序
2. **压力感知** - 根据上下文使用率动态调整策略
3. **智能剪枝** - 自动压缩工具输出，保留关键信息
4. **指数衰减** - 旧信息权重自动降低

### 4.2 核心升级点详解

#### 升级点 1：Core Memory 记忆块（来自 Letta）

**新增 Layer 0，将核心身份信息标签化、结构化：**

```yaml
# core_memory.yaml
human:
  name: "90"
  timezone: "Asia/Shanghai"
  risk_preference: "conservative"
  communication_style: "formal"
  
persona:
  name: "master-quant"
  role: "全局最高权限中枢"
  scope: "量化研究生态系统"
  unique_interface: "仅对接用户 + 14 个子智能体"

  red_lines:
    - "绝不自动交易"
    - "绝不修改实盘参数"
    - "绝不泄露敏感信息"

    - "收到报告 → 转发用户 → 派发下一任务"
    - "四重交付物检查 → 沉淀归档"
```

**优势：**
- 结构化存储，便于快速检索
- 标签化设计，支持多智能体共享
- 字符数限制，防止上下文膨胀

```python
class MemoryPressureMonitor:
    LEVELS = {
        "low": (0, 50),      # 正常运行
        "moderate": (50, 75), # 开始预警
        "high": (75, 90),     # 激进剪枝
        "critical": (90, 100) # 强制干预
    }
    
    def check_pressure(self, current_tokens, max_tokens):
        usage_percent = (current_tokens / max_tokens) * 100
        
        if usage_percent >= 90:
            return self.intervene_critical()
        elif usage_percent >= 75:
            return self.intervene_high()
        elif usage_percent >= 50:
            return self.warn_moderate()
        else:
            return "normal"
    
    def intervene_critical(self):
        # 强制压缩工作记忆
        # 归档低优先级内容
        # 触发会话重置建议
        pass
```

#### 升级点 4：HOT/WARM/COLD 分层 + 陈旧检测（来自 Cortex TMS）

| 层级 | 文件 | 策略 |

